# TypeScript + Nix (bun)

## Dependency management

Dependencies are fetched reproducibly via [bun2nix](https://github.com/nix-community/bun2nix). The flow:

1. `bun.lock` — bun's lockfile (text format), committed to the repo
2. `bun.nix` — generated by `bun2nix` CLI from `bun.lock`, committed to the repo
3. `fetchBunDeps { bunNix = ./bun.nix; }` — used in `default.nix` to create a Nix-compatible bun cache

### When dependencies change

After running `bun add/remove/update` (which updates `bun.lock`):

1. Regenerate `bun.nix`: `nix run github:nix-community/bun2nix -- -o bun.nix`
2. Verify: `nix build`

## Build

- `nix build` — full production build (client + server bundle)
- `nix run` — build and run
- Client is built with `bun run build` (Vite) inside the client workspace
- Server runs via `bun` (TypeScript execution without compile step)
- bun2nix's hook handles node_modules setup, shebang patching, and lifecycle scripts
