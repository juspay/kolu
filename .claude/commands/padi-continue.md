# Follow-up Changes

Make follow-up code changes on the current branch, then run through the remaining workflow steps (implement → e2e → fmt → commit → police → test → ci → update-pr → docs → done).

Use this when you need to make additional changes after the initial workflow run — fixes from code review, polish, additional features on the same PR, etc.

ARGUMENTS: $ARGUMENTS

Invoke: `/padi do --from followup -- $ARGUMENTS`
