With five terminals open on the canvas it's hard to tell at a glance which one is
*thinking*, which is *waiting for you*, and which just *pinged you and you missed
it*. That state already exists — it shows as a label in each tile's title bar
(`Thinking`, `Waiting for input`, `Running in background`) and as colored dots in
the dock. But reading a title bar isn't a glance, and you're looking at the
**terminals**, not the dock. So put the signal on the **tile border**, where
peripheral vision can catch it across the whole canvas.

<Callout kind="note" title="The ask">
Make running terminals visually distinct on the canvas. An **animating border**
for ones that are working, a **throbbing border** for ones with a fresh alert,
and **some other border** for ones that are waiting on you — so a sweep of the
eye tells you the state of the fleet without reading anything.
</Callout>

This note is a **chooser**: four animated border languages rendered live below,
each across the full attention ladder — which is *ranked* (some tiles matter more
than others) and *time-aware* (a "waiting" tile cools as it ages) — plus the
secondary decisions (color, motion, decay, settings). Pick a language and the open
decisions, and it ships on the `throbborder` branch.

## The attention ladder

The signal isn't a handful of flat colors — it's a **loudness ladder ordered by
who needs you**: `alert ▸ fresh-waiting ▸ working ▸ stale-waiting ▸ idle`. And
one rung isn't fixed — **"waiting" slides down the ladder as it ages**. A tile
that started waiting a minute ago should outrank a busy one; the same tile six
hours later has cooled below it. kolu already ranks and ages exactly this, so the
border reuses the machinery rather than inventing a parallel scheme:

| Rank | State | When | Color | Border |
| --- | --- | --- | --- | --- |
| **1 · Alert** | `unread` — flipped to needing you **while you weren't looking** | violet `--color-alert` | fast **throb** + halo (~1.2s); **self-clears on focus** |
| **2 · Waiting · fresh** | `awaiting` and recent — within your activity window (esp. < 4h) | violet `--color-alert` | gentle slow **breathe**, bright |
| **3 · Working** | thinking / running tools / background workflow | rust `--color-busy` | **steady** hum — no motion |
| **4 · Waiting · stale** | `awaiting` but older than your activity window (parked) | violet `--color-alert` | dim **ember**, static — cooled *below* the working hum |
| **5 · Idle / done** | no agent, finished, or acknowledged | repo identity | none |

<Callout kind="accent" title="Two refinements the priority order forces">
**1 · "Waiting" splits in two and *decays.*** A fresh waiter outranks a working
tile; the same tile, hours later, has cooled below it. The crossover is your
**activity window** — the `4h / 12h / 24h / All` control already in the sidebar
(<Cite file="packages/client/src/terminal/activityWindow.ts" />, default `24h`).
Pick `4h` and a waiter drops below the working hum after four hours; `All`
disables decay so every waiter stays fresh. The loudest rung (alert) still clears
itself the instant you focus the tile.

**2 · Motion means "needs you," not "busy."** Because alert (1) and fresh-waiting
(2) outrank working (3), the strongest attention cue — *motion* — is reserved for
the top of the ladder. Working is a **steady** glow; only the rungs that want you
actually move. This flips the first instinct (an *animating* border for working);
see **Open decisions** below.
</Callout>

## Waiting cools as it ages

This is the new axis. The same waiting tile, rendered at four ages — its violet
glow dims monotonically and stops moving as `lastActivityAt` recedes, crossing
*below* the rust working hum at your activity-window threshold and settling near
idle once it parks. The clock is kolu's existing `useStaleCheck()` /
`activityWindow` — the very signal that already parks dock rows and fades minimap
tiles, now cooling the canvas border too. One vocabulary, one persisted choice.

## The active tile — where focus meets the ladder

The ladder ranks *attention demand* — which tile to look at **next**. Focus marks
where you **are**. Different axes, and the active tile is where they meet, so make
the composition explicit:

- **Focus is its own channel, already built.** The active tile carries a teal
  inner ring + a right-edge accent (`store.activeId()` → the existing
  `0 0 0 1px var(--color-accent)`). Teal = "you are here"; the rust/violet aura =
  "this wants you." Two colors, two geometries — inner ring vs outer glow — so
  they don't collide, and rust-for-working keeps the hues apart.
- **The active tile is never *alert.*** Focusing a tile clears its `unread`
  (existing behavior — sound, OS badge, and pip all reset). So the loudest rung
  only ever lights an *inactive* tile: the eye-grab always points you somewhere
  **new**, never back at yourself.
- **Focus mutes the tile's own aura.** You're already attending to it, so it
  shouldn't compete — the `--aura-intensity` knob simply scales down on
  `data-active`. An active working tile is a clean teal ring with its rust aura
  dialed to a whisper; an active waiting tile keeps a faint violet (you can still
  see it wants input) while focus dominates.

**One compositional rule.** A tile's effective border loudness is
**`state-tier × age-decay × presence`**, where presence is the existing opacity
hierarchy (active `1.0` · inactive `0.92` · parked `0.55`) — a box-shadow aura
fades with its element's opacity for free. Active *additionally* mutes the aura;
parked is already dim. The product guarantees the **brightest border on the
canvas is always an inactive, fresh, attention-class tile** — precisely where the
eye should land. Focus and the ladder reinforce each other instead of fighting.

## Prototypes

Four border languages, each rendered live across the ladder (waiting shown
*fresh*). They differ in *which channel* carries state and *how energetic* it
reads — and two tiebreakers now matter: which can express a **continuous loudness
ramp** (so a waiter can dim), and which **survives scaling** to the minimap so the
canvas and its map speak one language.

### A — Halo (recommended)

<Pill variant="good">Recommended</Pill> A soft outer **glow** carries state; the
border line stays the repo identity color. A glow has a *continuous* brightness
axis, so it's the only language that renders the whole ranked, aging ladder
cleanly — alert throbs brightest, fresh-waiting breathes a bright violet, working
sits as a steady rust hum, and a stale waiter just **dims** toward an ember (the
decay strip above is literally this language turned down). It also leaves the two
existing border meanings — repo identity and the focus ring — untouched.

<Gallery lang="halo" lead="A · Halo — glow (idle ▸ working ▸ waiting·fresh ▸ alert)" />

### B — Live border

The **border line itself** takes the state color — the most literal reading of
"animated border." It *can* honor the rank (steady rust for working, a breathing
violet for fresh-waiting, a throb for alert), but two problems: it overwrites the
repo-identity color while an agent is busy (most of the time), and a solid line
has nowhere to **dim** to — a faint border reads as a rendering bug, not a cooled
waiter, so the decay ramp has no room. Fine for a binary; awkward for a ranked,
aging ladder.

<Gallery lang="border" lead="B · Live border — the line carries state" />

### C — Sweep

A comet of light **marches around the border** while working — the most "this is
actively running right now" of the set, echoing the dock rail's orbiting glow.
But that's the trap under the priority order: it spends the loudest channel
(continuous motion) on the **lowest-priority** active state — working, rank 3 —
pulling the eye to a churning tile over one that's actually blocked on you. The
rank inverts it. Keep it only if "what's running" matters more than "what's
waiting." (Attention states fall back to the halo throb.)

<Gallery lang="sweep" lead="C · Sweep — a marching comet for working" />

### D — Top bar (reads at any zoom)

A thin status **bar on the top edge** — steady rust for working, a gentle violet
pulse for fresh-waiting, a sharper blink for alert, nothing for idle. It rings
nothing and stays out of the title bar, so it's the calmest of the four. Its real
edge is **scale-invariance**: a colored top edge is the same legible shape on a
full canvas tile *or* a 40px minimap marker — which the others can't claim (a glow
or a marching comet dissolves when the tile is tiny).

<Gallery lang="bar" lead="D · Top bar — top-edge status strip" />

And the minimap already wants a state signal: today it paints a **corner dot**
(`bucketDescriptor().accentVar`). Language D **unifies** the two surfaces — the
same top edge, scaled down — so the canvas and its map speak one language:

## Open decisions

Independent of the language, a few choices shape how loud and how broad this gets.

### Working color — rust, not teal

The active/focused tile already owns **teal** (`--color-accent`: the focus ring +
the right-edge accent). If "working" is *also* teal, a focused-and-working tile is
teal-on-teal — is it busy, or is it the one I'm looking at? Coloring working

### Motion = needs-you (working stays steady)

The priority order mostly *answers* the earlier "should working animate?"
question: no — if a busy tile out-pulses a blocked one, the eye lands in the
wrong place. So motion is reserved for the rungs that want you (alert throbs,
fresh-waiting breathes) and **working is a steady hum** (as shown). The remaining
lever is taste: fully static working, or a *barely-there* breath so it still
reads "alive" — as long as it stays clearly quieter than fresh-waiting.

### The decay curve & threshold

How "waiting" cools is its own decision:
- **Continuous vs banded.** A smooth `--aura-intensity` from `age / threshold`
  (the strip above is hand-stepped to fake it), or reuse the existing **age
  bands** — `idleBucketFor()` already cuts `4h-12h / 12h-24h / 24h-48h / 48h+`
  (<Cite file="packages/client/src/terminal/activityWindow.ts" />) for discrete
  steps with zero new math.
- **Which window.** Reuse the persisted, user-chosen `activityWindow` (default
  `24h`); honor `All` = no decay. The one control already in the sidebar drives
  dock, minimap, *and* this.
- **Where stale lands.** A dim violet ember (still legibly "waiting"), or fully
  demote to idle/parked to match how the dock and minimap already treat parked
  rows. Ember keeps the "it wanted you" memory; parked is quieter.

### The active tile's aura — mute or suppress

The active tile already wins via the teal ring and can't be `alert` (focus clears
unread). The open call is the *rest* of the ladder on it: **mute to a whisper**
(~25–35% intensity, so you still glimpse working/waiting on the tile you're in),
or **fully suppress** (focus ring only, zero aura — calmest, leans hardest on
"borders are for the tiles you're *not* in"). Default: mute.

### Idle / done, settings, reduced motion

- **Idle / done** stays bare (repo border only) — don't add noise to the quiet
  ones. A separate "just finished" pulse is possible but is arguably the `alert`
  rung's job already.
- **Reduced motion** follows the existing convention — every gallery above
  collapses to a static high-opacity state under
  `prefers-reduced-motion: reduce`; nothing depends on motion alone to be
  readable.
- **Settings** — likely on by default (reduced-motion covers the calm case); a
  `PreferencesSchema` toggle is a small add if an explicit off-switch is wanted.

## How it wires in

Small and contained — no new state machinery, because the classifier already
exists and is the single source of truth.

- **Shared mapper.** Generalize the dock's <Cite file="packages/client/src/canvas/dock/pipVariant.ts" /> `(bucket, unread)` mapper into `auraVariant(bucket, unread, age) → { tier, intensity }` consumed by the tile *and* the dock, so they can never drift. The state→tier decision lives in exactly one place.
- **The clock is already built.** Age comes from `useStaleCheck()` / `activityWindowThresholdMs()` over `meta.lastActivityAt` (<Cite file="packages/client/src/terminal/staleness.ts" />, <Cite file="packages/client/src/terminal/activityWindow.ts" />) — the same reactive, 60s-ticking staleness the dock and minimap already use. No new threshold, no new timer.
- **The tile.** <Cite file="packages/client/src/canvas/CanvasTile.tsx" /> reads `agentBucket(meta.agent)` + `isUnread(id)` + the stale check (all in scope), sets a `data-aura` tier and a `--aura-intensity` custom property, and paints the aura via an **outer** channel (box-shadow glow) so the existing border (repo identity) and the inner focus ring (`0 0 0 1px var(--color-accent)`) are untouched — different geometry, no collision. When `props.active`, it scales `--aura-intensity` down (focus already won — and `unread` self-clears on activation, so the active tile is never the alert), keeping the teal ring dominant.
- **One language across surfaces (if D).** The minimap already classifies via `agentBucket` / `bucketDescriptor` (<Cite file="packages/client/src/canvas/CanvasMinimap.tsx" />) — the top-bar treatment swaps its corner dot for a top edge reading the same `auraVariant`, so the canvas tile and its minimap marker stay in lockstep off one mapper.
- **CSS.** One `@keyframes` block in <Cite file="packages/client/src/index.css" /> beside the existing `pill-*` / `dock-chip-*` animations, with the matching `prefers-reduced-motion` fallback.
- **Discoverability.** A tip per <Cite file=".claude/rules/conventions.md" /> (`settings/tips.ts`) so people learn the border means something.

Roughly one CSS block + ~20 lines in `CanvasTile` + a tiny shared function — and zero new state or timing machinery.
