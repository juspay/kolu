You asked for **Sleep**: leave a Claude Code terminal blocked for days, sleep it
(its PTY, xterm, WebGL context, and agent all released — gone, like closing it),
and **wake** it later in place with the agent resumed.

The model is one move: make the terminal a sum — **`Terminal = active | sleeping`**.
A dormant terminal is a *state* on the same record, so it inherits focus, drag,
dock ordering, and persistence for free — it *is* the terminal the canvas already
renders, just without a PTY. Four phased PRs; the first is a zero-behavior-change
foundation.

## A terminal is active or sleeping

The fold already lives in the schema. `surface.ts` splits a terminal's fields into
a **persisted base** (cwd · git · intent · theme · `canvasLayout` — survives a
restart) and a **live overlay** (agent · foreground · live-PR · the PTY/xterm/attach
handles — "never persisted; a restore must re-derive it"). That partition *is*
active-vs-sleeping, so the sum maps onto bases that already exist — exactly one
field is added:

```ts
const ActiveTerminalSchema =
  PersistedTerminalFieldsSchema.merge(LiveTerminalFieldsSchema)   // live overlay present
    .extend({ state: z.literal("active") });

const SleepingTerminalSchema =
  PersistedTerminalFieldsSchema                                   // base only — overlay absent by type
    .extend({ state: z.literal("sleeping"), sleptAt: z.number() });

const TerminalMetadataSchema =                                   // the wire / collection shape
  z.discriminatedUnion("state", [ActiveTerminalSchema, SleepingTerminalSchema]);

type Terminal =
  | ({ state: "active" }                    & PersistedTerminalFields & LiveTerminalFields)
  | ({ state: "sleeping"; sleptAt: number } & PersistedTerminalFields);

// Presence reads the union; touching a live field MUST narrow.
const placeTile  = (t: Terminal) => t.canvasLayout;             // both arms — no narrow
const routeInput = (t: Terminal) => {
  if (t.state !== "active") return;                             // compiler-forced narrow
  send(t /* now carries pr · agent · foreground + a live PTY */);
};
```

`sleptAt` is the sleeping arm's analogue of the live overlay — the only new scalar.
An active terminal is `base + overlay`; a sleeping terminal is `base + sleptAt`.
Sleeping is one record whose `state` flag says whether its PTY is currently spawned.

## Presence reads the union, liveness narrows

<Arch />

Putting the discriminant on the terminal buys two structural properties:

- **Type-safe presence vs liveness.** A consumer that reads `agent`/`foreground`/`pr`/PTY/xterm must first narrow `state === "active"`; the compiler refuses a live field on the bare union. There is no live-only list to read from by mistake — so a presence surface (canvas, dock, minimap, arrange, cycle, switcher) physically *cannot* omit or mis-render a sleeping terminal. Input routing resolves inside the active narrow, so a sleeping terminal can be the active/selected/panned-to tile yet is never an input target. `holdsWebgl`/`webglTileBudget` filter on the `active` set, so a sleeping terminal holds no WebGL context by construction.
- **One persistence channel.** `SavedTerminal` is already `PersistedTerminalFields + id` — the sleeping arm's exact payload minus `sleptAt`. A restored terminal and a slept terminal are the *same on-disk record*, distinguished only by `state`. Session save emits one list; sleeping terminals join it; restore is bulk-wake, wake is restore-one, sleep is the inverse.

## The phases

Each is one reviewable PR; each leaves `master` shippable.

| Phase | What lands | Why separable |
|---|---|---|
| **1 — Seat the sum** | Add `state`; flip `TerminalMetadataSchema` to a `discriminatedUnion`; presence surfaces read the `Terminal` union off the terminal store; ship with only the `active` arm constructed | **Zero behavior change** — a pure structural move that makes the narrowing seam exist before any sleep logic depends on it |
| **2 — Sleep / Wake** | Populate the `sleeping` arm: sleep snapshots the base + releases PTY/xterm/agent; wake respawns from the base; sleeping terminals join the canvas/dock as ordinary cards and rows | The user-facing core; no new component, no second loop |
| **3 — Frozen screenshot body** | Capture just before sleep, write under `KOLU_STATE_DIR`, serve through a small static image route; the reference rides the record | Isolated surface — one capture + one route |
| **4 — Unify with session-restore** | Collapse boot-restore and wake onto one seam (rehydrate a persisted terminal from its base); the session snapshot and sleeping records become one ordered set | Behavior-preserving consolidation, landed last |

### Phase 1 — seat the sum (zero behavior change)

Add the `state` discriminant to the terminal and flip `TerminalMetadataSchema`
from a flat `.merge` to `z.discriminatedUnion("state", …)`, but **construct only
the `active` arm**. Every terminal is `active`, so the UX is pixel-identical; what
the phase delivers is the `state === "active"` seam, compiling and in place. Presence
surfaces — the canvas `<For>`, dock ranking, the minimap, arrange-by-repo, the
`nextTerminal`/`prevTerminal` cycle, view-posture's count gate, the right-panel
`hasTerminals` check — read the `Terminal` union directly off the terminal store,
with no registry between them. The canvas card shell, the layout writer, and
arrange's clustering algorithm are untouched; only their data source changes.

Two things land here because the schema flip forces them:

- **The migration.** `SavedTerminal` is persisted, so the `discriminatedUnion`
  rejects a legacy record with no `state` key. A `state.ts` migration stamps
  `state: "active"` on every legacy terminal and bumps `SCHEMA_VERSION` — the *one*
  sanctioned place to supply the default. Read sites narrow; they never coalesce
  (a read-time `?? "active"` is a defect — fail fast).
- **The liveness-reader sweep.** Every reader that touches `agent`/`foreground`/`pr`/PTY/xterm
  (the agent indicator, the PR popover, the xterm pane, the WebGL budget, input
  routing, `activeMeta`) either stays in the presence subset or adds a
  `state === "active"` narrow. That sweep is the bulk of the diff — and it *is* the
  win: afterward a live field on a bare terminal won't compile.

Acceptance: `master` is shippable, the UX is identical, and the seam every later
phase leans on is in place.

### Phase 2 — Sleep / Wake

Populate the `sleeping` arm. **Sleep** snapshots the persisted base, then releases
the PTY/xterm/agent — *persist before kill*, never kill-then-pray, so a crash
mid-sleep loses nothing. **Wake** respawns from the base and resumes the agent. A
terminal transitions `active → sleeping` atomically on one record, so there is no
"live wins" dedup window and no cross-arm transition to reconcile. Sleeping
terminals join the one set the canvas and dock already iterate, so they are
focusable/draggable/resizable cards and selectable dock rows with **no new
component and no second loop** — only a dormant placeholder body (Phase 3 swaps in
the frozen frame) and one new verb, Wake. On boot they rehydrate *as sleeping*,
never auto-woken, and count toward `tileCount` so a sleeping-only workspace stays
on the canvas.

<Callout kind="warn" title="Non-negotiables for Phase 2">
These are requirements, not nice-to-haves — each closes a failure mode that is
easy to ship without noticing.

- **Snapshot the top terminal only.** A sleeping record is a single terminal —
  splits are released, not frozen. The record is rooted by construction (its id is
  the terminal it was), so no orphan record can arise.
- **Tolerate a malformed record; never let one poison the set.** Validate *shape*
  in the schema and enforce the cross-field invariant (the base is well-formed) by
  **dropping** a bad record at the read boundary — never a fatal validator on the
  persisted collection. The stakes are high because sleeping records share the
  terminal list: one corrupt record must not break the whole list. (See the
  `persisted-schema-stays-tolerant` code-police rule.)
- **First-class *and visually distinct* in every presence surface.** A sleeping
  terminal renders — moonlit, ☾, dimmed — in the canvas, dock, minimap, switcher,
  and mobile; arrange clusters it with its repo; the cycle traverses it. The sum
  makes omission unrepresentable, but verify it on each surface.
- **A sleeping tile can be the active tile.** Click/select focuses it; input
  routing narrows to `active`, so it is never an input target — a type fact, not a
  runtime guard.
- **An e2e that drives the real failure modes.** Seed a *malformed* record (it must
  not poison the set), sleep a terminal (it must not vanish), wake it (it must be
  usable), and assert sleeping appears on each presence surface — driven like a
  user, not asserted on counts. A green count-only test over a fresh state proves
  nothing here.
</Callout>

<Callout kind="good" title="Why the order is the point">
Sleep's machinery is small. The design work is *where a dormant terminal lives* —
Phase 1 answers that with a flag instead of a layer, and everything after is a
body, a route, or a consolidation. Small, because the structure finally fits the
thing.
</Callout>

## Trade-offs & when we'd revisit

- **A migration is mandatory.** `SavedTerminal` is persisted, so adding `state`
  needs the `state.ts` migration above. There is no shipping this as a pure
  refactor.
- **A wide type-level ripple is the diff.** Flipping `TerminalMetadataSchema` to a
  `discriminatedUnion` forces every liveness reader to either stay in the presence
  subset or add a narrow. That sweep *is* the win, but it is the bulk of the work.
- **`state` must be lifecycle-owned.** Only the sleep/wake transitions write
  `state`; a metadata provider must never resurrect a live field on a sleeping
  record (the overlay isn't on that arm — a type error, which is the point).

<Callout kind="accent" title="When we'd revisit">
If a genuine *non-terminal* canvas kind ever lands — a sticky note, a web embed —
introduce a content-kind layer then, designed against **two real kinds**. Location
is structure: the receptacle is justified the day there's a second thing to put in
it, and not before. (See `electricity.mdx` — the same trap, named three ways.)
</Callout>
