You asked for **Sleep**: leave a Claude Code terminal blocked for days, sleep it
(its PTY, xterm, WebGL context, and agent all released — gone, like closing it),
and **wake** it later in place with the agent resumed.

This is the plan's **second** re-grounding, and it reverses the first. The first
re-plan diagnosed the blocker as "there is no first-class *tile* in kolu — a tile
*is* a live terminal", and fixed it by building a Tile registry (<PrLink pr={1435} />,
merged) with a `TileContent` union, intending sleeping to be a second content
*kind*. Building sleeping on it (<PrLink pr={1441} />) surfaced the deeper truth:
**sleeping is a *state* of a terminal, not a kind of canvas content** — and the
tile layer we added to host that "second kind" hosts a phantom, because the kind
never materialized. So we close <PrLink pr={1441} />, **collapse** the tile layer,
and move the discriminant where it belongs: onto the terminal.

## The pivot — sleeping is a terminal state

A first-class tile would earn its keep the day the canvas holds something that
*isn't* a terminal (a sticky note, a web view). It doesn't, and it isn't on the
near roadmap — so a content-kind layer above the terminal is **speculative
generality**: a one-armed union whose only "second kind" is a mis-levelled state.
Both the data-loss bug and the "wrong store" bug class on <PrLink pr={1441} /> trace
to that mis-levelling and to its enabling alias, `TileId = TerminalId`.

The clean model puts the sum on the **terminal** itself — and the fold already
exists in the schema. `surface.ts` splits a terminal's fields into a **persisted
base** (cwd · git · intent · theme · `canvasLayout` — survives a restart) and a
**live overlay** (agent · foreground · live-PR · the PTY/xterm/attach handles —
"never persisted; a restore must re-derive it"). That partition *is*
active-vs-sleeping:

```ts
// surface.ts bases UNCHANGED — the fold already exists; exactly one field is added.
const ActiveTerminalSchema =
  PersistedTerminalFieldsSchema.merge(LiveTerminalFieldsSchema)   // live overlay present
    .extend({ state: z.literal("active") });

const SleepingTerminalSchema =
  PersistedTerminalFieldsSchema                                   // base only — overlay absent by type
    .extend({ state: z.literal("sleeping"), sleptAt: z.number() });

const TerminalMetadataSchema =                                   // the wire/collection shape
  z.discriminatedUnion("state", [ActiveTerminalSchema, SleepingTerminalSchema]);

type Terminal =
  | ({ state: "active" }                    & PersistedTerminalFields & LiveTerminalFields)
  | ({ state: "sleeping"; sleptAt: number } & PersistedTerminalFields);

// Reading: presence reads the union; touching a live field MUST narrow.
const placeTile  = (t: Terminal) => t.canvasLayout;             // both arms — no narrow
const routeInput = (t: Terminal) => {
  if (t.state !== "active") return;                             // compiler-forced narrow
  send(t /* now carries pr · agent · foreground + a live PTY */);
};
```

`sleptAt` is the sleeping arm's analogue of the live overlay — the only new
scalar. An active terminal is `base + overlay`; a sleeping terminal is
`base + sleptAt`. Sleeping is one record whose `state` flag says whether its PTY
is currently spawned.

The registry's whole stated job was to keep tile **presence** (exists, sits on
the canvas, focusable, has a dock row) apart from terminal **liveness** (PTY ·
xterm · agent · attach). But it does that by *indirection over a fold the schema
already owns*: `tileIds` is a verbatim re-export of `terminalIds()`, the
selection signals are verbatim re-exports of view-state, `contentOf` fabricates a
`{kind:"terminal",terminalId:id}` object on every read, and `getLayout/setLayout`
are a one-kind dispatch. Move the discriminant onto the terminal and the layer
has nothing left to do: presence consumers read the union, liveness consumers
narrow, and there is no second store, no synthesized content object, and no
separate dormant cell to merge.

## The surgical collapse

It isn't a `git revert` — the genuinely-good canvas primitives stay; only the
presence-vs-liveness *bridge* goes. Tellingly, the two keystones <PrLink pr={1435} />
advertised don't even live in `tile/`: the reference-stability memo is in
`useTerminalMetadata.ts`, the selection signals in `useViewState.ts`. So the
**entire `tile/` directory is drop**, and every keep either already lives outside
it or only needs its import edge re-pointed from `tileStore.*` to `store.*`.

| Keep | Why it stays |
|---|---|
| `CanvasTile.tsx` — the content-agnostic card shell (drag/resize, title bar, pan/zoom transform, mode union, aura/on-screen gating) | A real UI primitive keyed on `id: string` + render-prop slots; knows nothing of terminals. The #988 wrapper-less transform is load-bearing canvas mechanics the sum never touches. |
| `TerminalCanvas.tsx` spatial machinery — `pendingLayouts`, placement cascade, drag/resize commit + grid-snap, focus/center effects, minimap drag | Pure geometry over `tileIds: string[]` + a layout seam; survives unchanged, only the seam re-points at the terminal store. |
| `sameTerminalIdOrder` + the `terminalIds()` memo (#1425 keystone) | Already lives in `useTerminalMetadata.ts`; the re-export is what's deleted. The metadata-only-tick equals gate keeps suppressing canvas/dock/mode churn. |
| `persistCanvasLayout.ts`; `useCanvasArrange.ts` domain logic | Dependency-light leaves / genuine canvas behavior. Algorithms untouched; plumbing re-points off `tileStore.*`. |

| Drop | Replaced by |
|---|---|
| `tile/useTileStore.ts` — the registry singleton | The terminal store itself, keyed by the `Terminal` union. Presence-vs-liveness is a `state === "active"` narrow, not a second store. |
| `tile/tileContent.ts` — `TileContent` peer union + `TileId = TerminalId` alias | The discriminant on the terminal (`state`). No separate id space — terminal ids are canvas ids; the `kind:"terminal"` arm carried no information. |

| Re-exposed `activeId`/`activate`/`setActiveSilently`/`tileCount` | Read view-state directly; `tileCount()` becomes `terminalIds().length` (counting sleeping). The wrong-store bug class disappears at the type level, so the re-export buys nothing. |

## The phases

Each is one reviewable PR; each leaves `master` shippable.

| Phase | What lands | Why separable |
|---|---|---|
| **1 — Collapse + seat the sum** | Delete `tile/`; re-point every consumer (canvas `<For>`, dock, minimap, arrange, view-posture, commands, right-panel) at the terminal store; add the `state` discriminant but ship with only the `active` arm populated | **Zero behavior change** (every terminal is `active`) — a pure structural move that makes the narrowing seam exist and compile before any sleep logic does |
| **2 — Sleep / Wake core** | Populate the `sleeping` arm: sleep = snapshot the base + release PTY/xterm/agent (reuse kill-with-subs); wake = respawn from the base (reuse restore). Sleeping terminals join the one set the canvas/dock iterate — focusable/draggable tiles, selectable rows — with a dormant placeholder body and one Wake verb. Rehydrate-as-sleeping on boot | The user-facing core; the dedup-window and cross-arm-transition awkwardness are gone *by construction* |
| **3 — Frozen screenshot body** | Capture via the existing `screenshotTerminal` path just before sleep, write under `KOLU_STATE_DIR`, serve through a small static image route (serve-dir / iframe-preview precedent); the reference rides the record | Isolated surface — one capture + one route; swaps Phase 2's placeholder for a real frame |
| **4 — Unify with session-restore** | Collapse boot-restore and wake onto one seam: "rehydrate a persisted terminal by respawning its PTY from the base." Boot = bulk wake, wake = activate-one, sleep = the inverse; the disjoint sleeping store and the session snapshot merge into one ordered terminal set | Behavior-preserving consolidation, landed last, designed against three concrete callers instead of two hypothetical ones |

## Payoff & trade-offs

Two wins fall out of putting the discriminant on the terminal:

- **Type-safe presence vs liveness.** A consumer that touches pr/agent/foreground/PTY must first narrow `state === "active"` — the compiler refuses a live field on the bare union. The "wrong store" bug class (<PrLink pr={1441} />'s minimap/mobile/cycle/arrange reading a live-only list) becomes *unrepresentable*, because there is no live-only list. The same narrow relocates `focusedId()` (the split-focus divergence is real and stays, but input-routing resolves *inside* the active narrow) and re-points `holdsWebgl`/`webglTileBudget` at the `active` set — a sleeping tile sits in the MRU and on the canvas yet holds no WebGL context, by construction.
- **Session-restore unification.** `SavedTerminal` is already `PersistedTerminalFields + id` — the sleeping arm's exact payload minus `sleptAt`. A boot-restored terminal and a slept terminal become the *same on-disk record*, distinguished only by `state`. One persistence channel; restore = bulk wake, wake = restore-one, sleep = the inverse.

And the costs, named so they're chosen, not discovered:

- **A migration is mandatory.** `SavedTerminalSchema` is persisted, so adding `state` needs a `state.ts` migration defaulting legacy records to `"active"` + a `SCHEMA_VERSION` bump — the `discriminatedUnion` rejects a record with no `state` until the migration stamps it. That is the *one* sanctioned place to supply the legacy default; **read sites must narrow, never coalesce** (no read-time `?? "active"`).
- **A wide type-level ripple is the diff.** Flipping `TerminalMetadataSchema` from a flat `.merge` to a `discriminatedUnion` forces every liveness reader to either stay in the presence subset or add a narrow. That sweep *is* the win, but it is the bulk of the work.
- **`state` must be lifecycle-owned.** Only the sleep/wake transitions write `state`; a metadata provider must never resurrect a live field on a sleeping record (the overlay isn't on that arm — a type error, which is the point).

<Callout kind="accent" title="When we'd revisit">
If a genuine *non-terminal* canvas kind ever lands — a sticky note, a web embed —
reintroduce a content-kind layer, but designed against **two real kinds**, not the
one-armed speculative union we're removing now. Location is structure: the
receptacle is justified the day there's a second thing to put in it, and not
before. (See `electricity.mdx` — the same trap, named three ways.)
</Callout>
