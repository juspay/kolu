## User-facing description

<RemoteCanvas />

You switch hosts from the **ChromeBar** ‚Äî *kinda like tmux sessions* ‚Äî and a remote host's terminals sit on the canvas **beside** your local ones. A remote tile wears an `ssh` badge and carries the *same* live awareness a local tile does ‚Äî git branch/dirty, PR status, agent state, and a **green dot** when it's moving bytes ‚Äî and it survives a network blip or a kolu-server deploy with full scrollback. One local host is always present; remotes are added on demand, auto-detected from your ssh config.

## Architecture-level changes

<Svg svg={arch} caption="The multi-host shape. Every host runs the same hashed kaval daemon; kolu-server keys its terminal endpoint by hostId behind one TerminalEndpoint seam. The local driver (R2, shipped) spawns + supervises a kaval over a unix socket; the ssh driver (R8) reaches + provisions the same kaval closure over ssh stdio; the fold (R8) mirrors a host-side arivu's awareness + fs/git. Local awareness runs in-process; only remote crosses a process boundary." />

**One endpoint, bound to a dialed kaval.** A kaval is a daemon you *dial* ‚Äî local over a unix socket, remote over ssh ‚Äî so remote is **not a second backend**. There is *one* `TerminalEndpoint`, bound to a `HostLocation`-resolved kaval endpoint (`{kind:"local"}` today, `{kind:"remote",host}` in R8). That resolver is the **sole** place a tile maps to its kaval; everything downstream talks to the backend and never asks "which kind?". No `RemoteTerminalEndpoint` ‚Äî R8 adds the ssh *driver* as an additive sibling of R2's local driver, never a retrofit.

Two volatility axes stay open (R1/R2 closed the rest):

| Axis ‚Äî changes for its own reason | Encapsulated by |
| --- | --- |
| Where a terminal's state lives ‚Äî this machine vs an ssh host | the `HostLocation`-resolved kaval endpoint ‚Äî one impl + a discriminator, not a second backend |
| How the backend reaches its agent ‚Äî transport ¬∑ framing ¬∑ reconnect | `HostSession` + `@kolu/surface/links/stdio` ‚Äî unix socket today, ssh in R8 |

**Zero domain knowledge crosses the transport** (Hickey's cut): the agent runs the unmodified sensors and the boundary ships their *values*. An earlier per-domain design (`RemoteGitInfoProvider`, ‚Ä¶) was transport adapters wearing domain costumes; the single-endpoint seam dissolved them.

<Callout kind="warn" title="The awareness fold is domain logic, not electricity">
R8's fold merges each remote `AwarenessValue` into the **co-owned** `terminalMetadata` entry and **splits** it across the persisted/live fence ‚Äî `pr¬∑agent¬∑foreground` through the quiet live mutator, `cwd¬∑git¬∑lastAgentCommand¬∑lastActivityAt` through the one that arms autosave. Lose the split and you either re-arm the agent-firehose (a perf regression) or stop persisting. It speaks the whole terminal domain *and* the autosave fence, so it stays **inline in kolu-server** ‚Äî never a `@kolu/*` package.
</Callout>

**Already paid for, so R5‚ÄìR9 are narrow.** `@kolu/surface-nix-host` ships `HostSession`, `provisionAgent` (ships the *derivation* ‚Äî a darwin parent drives a linux remote with no cross-builder), and `resolveSystem`; `@kolu/surface` ships `mirrorRemoteSurface` (the per-primitive mirror). R2 already shipped the host-count-agnostic spine: `@kolu/surface-daemon-supervisor` and the per-connect `system.version` handshake. So R8 is a driver + fold behind shipped seams; R7 completes the surface mirror; R6 grows arivu's fs/git; R5 is one wire signal.

### R5 ‚Äî overflow-recovery loop

~130 LoC w/ tests; the next step. kaval sheds a slow attach subscriber by *ending* its iterator, today indistinguishable on the wire from a PTY exit, so the client treats the drop as terminal and freezes scrollback (`onOverflow` exists but is unwired). Thread a typed *overflow ‚Üí re-attach* signal through the attach contract so the client re-attaches for a **fresh snapshot** (reusing the existing `onRetry` reset), distinct from PTY-exit. **Ships independently of the remote work**: a slow subscriber is reproducible on the *local* socket, so it is CI-testable today, and the fix hardens the attach contract for *both* transports ‚Äî a clean prerequisite for the remote reconnect R8 leans on. [herdr vs kolu](herdr-vs-kolu.html) flagged the same gap.

<a id="r6"></a>

### R6 ‚Äî `@kolu/terminal-workspace`: one library, one fs/git impl, two homes

**Shipped** ‚Äî <PrLink pr={1506} />. The host-side library that knows a terminal's working directory becomes **`@kolu/terminal-workspace`** ‚Äî the shipped `@kolu/terminal-awareness` (R4.1) grown to cover fs/git too, run *in-process* locally (kolu) and hosted by *arivu* remotely: the same two homes the sensors already proved. R6 ships **one fs/git impl** (`createTerminalWorkspaceEndpoint`), *not* one surface both homes already serve ‚Äî the two homes re-expose that one impl through two **deliberately different** contract shapes:

- `awareness` (collection) + `activity` (stream) ‚Äî the derived status badge + the green-dot liveness, served on `terminalWorkspaceSurface`. **Shipped.**
- the Code tab's fs/git reads (`listAll` ¬∑ `readFile` ¬∑ `getStatus` ¬∑ `getDiff` + repo/file change watching) ‚Äî **new in R6**, lifted from kolu-server's `localEndpoint` (a thin wrapper over `kolu-git` + `serve-dir`) into the shared library. kolu-server (in-process) re-exposes them on `koluSurface`'s **value-bearing streams**; arivu (remote) serves them on `terminalWorkspaceSurface` as **procedures + `{seq}` pulse watcher streams** a consumer requeries on.

This kills the duplication wart: without it, arivu would **reimplement** fs/git and kolu would carry *two* fs/git impls (its own local, arivu's remote). Instead there is **one** ‚Äî in-process locally, via arivu remotely. Only the thin fs/git wrapper moves; the terminal-endpoint *orchestration* around it (spawn ¬∑ adopt ¬∑ the registry) stays kolu's and gets a remote impl in R8. The single shared **surface** both homes serve is *not* yet true ‚Äî it arrives in **R8**, which mirrors `terminalWorkspaceSurface` whole (via R7's total mirror), folding the `awareness` collection into `terminalMetadata` and *forwarding* the fs/git procedures. Until then the two contract shapes are kept deliberately distinct (R8's remote kolu re-queries procedures rather than streaming full diffs over the wire). Nothing is folded into `AwarenessValue` ‚Äî procedures were never part of it.

<a id="r7"></a>

### R7 ‚Äî complete `mirrorRemoteSurface` to a total dual

`mirrorRemoteSurface` is the consume-side **dual of `implementSurface`** ‚Äî and a dual must mirror *every* primitive its producer wires. `implementSurface` wires procedures too, but the mirror's `SurfaceSink` carries only `cells`/`collections`/`streams`/`events`: a **leaky half-mirror**, harmless only because every surface mirrored so far had **zero procedures**. R7 closes the gap: the mirror maps **every** primitive by its kind ‚Äî streaming state ‚Üí a **sink** (the consumer folds/aggregates), procedures ‚Üí a **forwarding stub** (a local call runs on the remote and returns; nothing copied). Then `mirrorRemoteSurface(remote)` yields a **local surface that *is* the remote one** ‚Äî `serve ‚àò mirror ‚âà identity`, the location-transparency the whole epic rests on. A `@kolu/surface` change ‚Üí the **drishti merge-gate**.

**Proven by drishti ‚Äî the canonical external second consumer.** [drishti](https://github.com/srid/drishti) is the browser ‚Üî Bun ‚Üî ssh process-monitor that already mirrors per-host *process* surfaces (the graduation that first proved `@kolu/surface` is electricity), and it is **read-only today**. R7's proof is drishti gaining a **"Kill process"** action ‚Äî the first *forwarded procedure* on a mirrored surface, exercised end-to-end (browser ‚Üí Bun ‚Üí ssh ‚Üí remote agent ‚Üí `kill(pid)`), **zero kolu**, riding the same drishti gate every `@kolu/surface` change already passes. Because that proof lives entirely in drishti's own surface, **R7 stands alone ‚Äî it needs neither R6 nor kolu**; R8 then consumes a *proven* total mirror for arivu's fs/git, not a speculative one.

<a id="r8"></a>

### R8 ‚Äî kolu dials remotes + the fold

The linchpin ‚Äî it **needs R6** (the shared fs/git library) **and R7** (the total mirror). kolu-server gains the **N-endpoint registry + the ssh driver** ‚Äî the `{kind:"remote",host}` arm, the `(host,id)` join, a persisted `location` ‚Äî reusing the *same* `getHostSession` the spike (R3) built. It dials the host's **kaval** for the PTY (via the shipped `frontDaemonOverStdio`) ‚Üí a remote tile; and dials its **arivu** over a *long-lived* `HostSession`, **mirroring the whole `terminal-workspace` surface** (R7) into one local handle. Because the mirror is total, kolu binds the remote `TerminalEndpoint` to that single handle and runs the **same** logic as against the in-process local surface ‚Äî never asking "which kind?":

- the `awareness` collection + `activity` stream are subscribed into kolu's **sink**, which *folds* each `AwarenessValue` into the tile's co-owned `terminalMetadata` across the persisted/live fence (see the callout) and lights the **green dot**;
- the fs/git **procedures** (`fsReadFile`/`gitDiff` + the watcher deltas) are **forwarded** on demand for the Code tab.

`location` is co-owned with `terminalMetadata`, persisted kolu-side, re-validated on reconnect against the remote kaval's live `cwd`. (Paste/upload is a `TerminalEndpoint` concern ‚Äî served behind the same seam with the `PRECONDITION_FAILED` guard ‚Äî not awareness, not fs/git.) The result: a remote terminal as capable as a local one ‚Äî PTY, awareness badge, green dot, working Code tab. The shape is the **lead diagram** above.

**Adoption ‚Äî a dial joins, never clobbers.** If the host already runs a kaval it is **adopted** (the same `adoptOrEnsure` spine the local boot uses); a duplicate is impossible by construction (the pid-gate's atomic `link(2)` EEXIST). On dial:

| the remote kaval is‚Ä¶ | kolu does |
| --- | --- |
| absent | provision + spawn fresh (`adoptOrEnsure` ‚Üí ensure) |
| live ¬∑ wire-compatible | **adopt** ‚Äî connect, never kill; its PTYs reconcile in |
| live ¬∑ a build behind | adopt **+** an "update pending" nudge (orthogonal) |
| live ¬∑ contract-skewed | recycle (kill ‚Üí respawn) ‚Äî only on a typed `DaemonContractSkewError` |
| unreachable (bad dial) | bounded retry, then **degraded ‚Äî never killed** |

One kaval per remote host: dialing *joins* its terminals ("the prod terminals"), never racing to kill them.

<a id="r9"></a>

### R9 ‚Äî canvas multiplex + validate

The UI: an active-kaval strip (local always present), a per-tile host hint, an ssh-config host picker ‚Äî *kinda like tmux sessions*. Then **validate the feel** on real use: does N-kavals-at-once beat tmux-style one-at-a-time switching? Fall back to switch if multiplex doesn't earn its keep. The granularity call lands here, validated, not up front.
