## User-facing description

<RemoteCanvas />

You switch hosts from the **ChromeBar** ‚Äî *kinda like tmux sessions* ‚Äî and a remote host's terminals sit on the canvas **beside** your local ones. A remote tile wears an `ssh` badge and carries the *same* live awareness a local tile does ‚Äî git branch/dirty, PR status, agent state, and a **green dot** when it's moving bytes ‚Äî and it survives a network blip or a kolu-server deploy with full scrollback. One local host is always present; remotes are added on demand, auto-detected from your ssh config.

## Architecture-level changes

<Svg svg={arch} caption="The multi-host shape. Every host runs the same hashed kaval daemon; kolu-server keys its terminal endpoint by hostId behind one TerminalEndpoint seam. The local driver (R2, shipped) spawns + supervises a kaval over a unix socket; the ssh driver (R9) reaches + provisions the same kaval closure over ssh stdio; the remote mirror (R9) reaches a host-side pulam's awareness + fs/git, composed into one surface (R8). Local awareness runs in-process; only remote crosses a process boundary." />

**One endpoint, bound to a dialed kaval.** A kaval is a daemon you *dial* ‚Äî local over a unix socket, remote over ssh ‚Äî so remote is **not a second backend**. There is *one* `TerminalEndpoint`, bound to a `HostLocation`-resolved kaval endpoint (`{kind:"local"}` today, `{kind:"remote",host}` in R9). That resolver is the **sole** place a tile maps to its kaval; everything downstream talks to the backend and never asks "which kind?". No `RemoteTerminalEndpoint` ‚Äî R9 adds the ssh *driver* as an additive sibling of R2's local driver, never a retrofit.

Two volatility axes stay open (R1/R2 closed the rest):

| Axis ‚Äî changes for its own reason | Encapsulated by |
| --- | --- |
| Where a terminal's state lives ‚Äî this machine vs an ssh host | the `HostLocation`-resolved kaval endpoint ‚Äî one impl + a discriminator, not a second backend |
| How the backend reaches its agent ‚Äî transport ¬∑ framing ¬∑ reconnect | `HostSession` + `@kolu/surface/links/stdio` ‚Äî unix socket today, ssh in R9 |

**Zero domain knowledge crosses the transport** (Hickey's cut): the agent runs the unmodified sensors and the boundary ships their *values*. An earlier per-domain design (`RemoteGitInfoProvider`, ‚Ä¶) was transport adapters wearing domain costumes; the single-endpoint seam dissolved them.

<Callout kind="note" title="The awareness 'fold' was never domain logic ‚Äî R8 deletes it">
Through R7, remote awareness was *folded* server-side into the **co-owned** `terminalMetadata` record, split across a persisted/live **fence** (`pr¬∑agent¬∑foreground` live; `cwd¬∑git¬∑‚Ä¶` persisted) whose only job was keeping the ~150 ms agent firehose off the session autosave. Both were artifacts of *one co-owned record*. R8 gives observation its own home: the sensors write `terminalWorkspaceSurface.awareness`, and kolu (and the browser) **read** it through one seam (`awarenessFor`) instead of folding a copy in. One writer per fact ‚Äî so the fold, the co-owned record, *and* the fence all **vanish** (the firehose never touches kolu). Detail + trade-offs in **R8**.
</Callout>

**Already paid for, so R5‚ÄìR10 are narrow.** `@kolu/surface-nix-host` ships `HostSession`, `provisionAgent` (ships the *derivation* ‚Äî a darwin parent drives a linux remote with no cross-builder), and `resolveSystem`; `@kolu/surface` ships `mirrorRemoteSurface` (now a total dual). R2 already shipped the host-count-agnostic spine: `@kolu/surface-daemon-supervisor` and the per-connect `system.version` handshake. So R9 is an ssh driver behind shipped seams; R8 composes one surface (the fold dissolves); R7 *completed* the surface mirror (<PrLink pr={1505} />); R6 grew pulam's fs/git (<PrLink pr={1506} />); R5 is one wire signal.

### R5 ‚Äî overflow-recovery loop

<Phase id="R5" name="overflow-recovery loop" status="todo" />
*Independent ‚Äî needs nothing, blocks nothing; landable any time.*

~130 LoC w/ tests; the next step. kaval sheds a slow attach subscriber by *ending* its iterator, today indistinguishable on the wire from a PTY exit, so the client treats the drop as terminal and freezes scrollback (`onOverflow` exists but is unwired). ### R6 ‚Äî `@kolu/terminal-workspace`: one library, one fs/git impl, two homes

<Phase id="R6" name="terminal-workspace ‚Äî one fs/git impl, two homes" status="shipped" needs={["R4"]} blocks={["R8"]} links={[{ label: "#1506", href: "https://github.com/juspay/kolu/pull/1506" }]} />

**Shipped** ‚Äî <PrLink pr={1506} />. The host-side library that knows a terminal's working directory becomes **`@kolu/terminal-workspace`** ‚Äî the shipped `@kolu/terminal-awareness` (R4.1) grown to cover fs/git too, run *in-process* locally (kolu) and hosted by *pulam* remotely: the same two homes the sensors already proved. R6 ships **one fs/git impl** (`createTerminalWorkspaceEndpoint`), *not* one surface both homes already serve ‚Äî the two homes re-expose that one impl through two **deliberately different** contract shapes:

- `awareness` (collection) + `activity` (stream) ‚Äî the derived status badge + the green-dot liveness, served on `terminalWorkspaceSurface`. **Shipped.**
- the Code tab's fs/git reads (`listAll` ¬∑ `readFile` ¬∑ `getStatus` ¬∑ `getDiff` + repo/file change watching) ‚Äî **new in R6**, lifted from kolu-server's `localEndpoint` (a thin wrapper over `kolu-git` + `serve-dir`) into the shared library. kolu-server (in-process) re-exposes them on `koluSurface`'s **value-bearing streams**; pulam (remote) serves them on `terminalWorkspaceSurface` as **procedures + `{seq}` pulse watcher streams** a consumer requeries on.

This kills the duplication wart: without it, pulam would **reimplement** fs/git and kolu would carry *two* fs/git impls (its own local, pulam's remote). Instead there is **one** ‚Äî in-process locally, via pulam remotely. Only the thin fs/git wrapper moves; the terminal-endpoint *orchestration* around it (spawn ¬∑ adopt ¬∑ the registry) stays kolu's and gets a remote impl in R9. The single shared **surface** both homes serve is *not* yet true ‚Äî it arrives in **R8**, which composes `terminalWorkspaceSurface` into the one surface kolu serves the browser (the server-side fold dissolving with it); **R9** then dials remotes and points that surface's per-terminal dispatch at the mirror. Until then the two contract shapes are kept deliberately distinct. Nothing is folded into `AwarenessValue` ‚Äî procedures were never part of it.

### R7 ‚Äî `mirrorRemoteSurface` is now a total dual ‚Äî *shipped* <PrLink pr={1505} />

<Phase id="R7" name="mirrorRemoteSurface ‚Üí total dual" status="shipped" blocks={["R9"]} links={[{ label: "#1505", href: "https://github.com/juspay/kolu/pull/1505" }]} />

`mirrorRemoteSurface` is the consume-side **dual of `implementSurface`** ‚Äî and a dual must mirror *every* primitive its producer wires. `implementSurface` wires procedures too, but the mirror's `SurfaceSink` carried only `cells`/`collections`/`streams`/`events`: a **leaky half-mirror**, harmless only because every surface mirrored so far had **zero procedures**. R7 closed the gap by kind: **streaming state stays PUSH** (subscribed and folded into the sink); a **procedure is PULL** (a local call runs on the remote and returns), so it can't sit in the push-only sink ‚Äî it comes back as a **forwarding stub**. The return is therefore `{ procedures, done }`: `procedures.<ns>.<verb>(input)` relays to the remote (total ‚Äî every procedure gets a stub, since a passthrough has no standing cost), and `done` is the former settle-on-link-close promise. Graft the stubs into a re-serve and `serve ‚àò mirror ‚âà identity` ‚Äî the location-transparency the whole epic rests on ‚Äî **pinned in-repo** by a hermetic unit test (`implementSurface` ‚Üí `directLink` ‚Üí mirror ‚Üí re-serve ‚Üí call). A `@kolu/surface` change ‚Üí the **drishti merge-gate**.

**Proven by drishti ‚Äî the canonical external second consumer.** [drishti](https://github.com/srid/drishti) is the browser ‚Üî Bun ‚Üî ssh process-monitor that already mirrors per-host *process* surfaces (the graduation that first proved `@kolu/surface` is electricity), and it was **read-only**. R7's external proof is drishti gaining a **"Kill process"** action ‚Äî the first *forwarded procedure* on a mirrored surface, exercised end-to-end (browser ‚Üí Bun ‚Üí ssh ‚Üí remote agent ‚Üí `kill(pid)`), **zero kolu**. That proof is the **merge-gate** for this kolu PR, not optional decoration: the return shape changed from `Promise<void>` to `{ procedures, done }`, and drishti `master` still `await`s the old shape on its streaming sinks ([drishti #70](https://github.com/srid/drishti/pull/70) adopted the #1497 `mirrorRemoteSurface`, all CI green). So the paired [drishti R7 PR #71](https://github.com/srid/drishti/pull/71) ‚Äî audit that `await` to `.done`, pin drishti's kolu source to this R7 revision, go green ‚Äî is now **open and gates #1505**: its full CI is green against the new `{ procedures, done }` API ‚Äî all 18 odu nodes (every lane on both `aarch64-darwin` and `x86_64-linux`), the "Kill process" forwarded-procedure action landing on top of the migrated mirror ‚Äî exactly the lane set #70 passed to gate #1497. Because that proof lives entirely in drishti's own surface, **R7 stands alone ‚Äî it needs neither R6 nor kolu**; R9 then consumes a *proven* total mirror for the remote dial. **One caution this plan earlier blurred:** the mirror (remote ‚Üí local, proven here by drishti) is a *different primitive* from **kolu composing-and-serving a stream to the browser** over `websocketLink` + `surfaceClient`. That second leg is **R-compose-2's**, and it is proven separately by **[pulam-web](pulam-web.html)** ‚Äî not by this mirror. drishti proved the mirror; *pulam-web* proves the browser leg.

### R8 ‚Äî compose terminal-workspace into kolu-server (the fold vanishes)

<Phase id="R-compose-1" name="awareness ‚Äî kolu reads (one writer per fact)" status="build-clean" needs={["R6"]} blocks={["R9"]} links={[{ label: "wake bug #1529", href: "https://github.com/juspay/kolu/issues/1529" }]} />
<Phase id="R-compose-2" name="git-status stream over ws" status="todo" needs={["R6", "R-pulamweb-4"]} blocks={["R9"]} links={[{ label: "repaint prep #1534", href: "https://github.com/juspay/kolu/issues/1534" }, { label: "deselect #1533", href: "https://github.com/juspay/kolu/issues/1533" }]} />

R6 left **one impl, two contract shapes**: kolu re-exposes fs/git on `koluSurface`'s own streams (in-process), pulam serves `terminalWorkspaceSurface` (remote). R8 collapses that to **one surface** ‚Äî and settles the question the epic kept reopening, *who owns awareness?* The answer is **not kolu**.

**Two moves, very different risk** ‚Äî a prototype (<PrLink pr={1510} />, *discarded as a branch*) built both and reached **full green CI**, which tells us a lot. **R-compose-1 ‚Äî awareness:** the one-writer-per-fact compose below. #1510 implemented it cleanly and it passed end-to-end, so it's **validated ‚Äî a clean build from a working reference**, not an open question. **R-compose-2 ‚Äî git-status over the ws:** moving the Code tab's git status onto `terminalWorkspaceSurface`'s `{seq}` watcher **stream**. #1510's months-long fight here turned out **not** to be the transport (the pulse reached the browser fine) but the **file-tree renderer** not repainting under change-pulse churn ‚Äî detail in its subsection. R-compose-2 still goes through [pulam-web](pulam-web.html)'s standalone graduation first (the drishti discipline), now sharpened to prove a *live DOM repaint*, not just data delivery. The architecture below is **R-compose-1**; R-compose-2 is its own subsection at the end.

A terminal is two things that were never one record:

- **authored state** ‚Äî kolu writes it, it is durable: id ¬∑ restore-cwd ¬∑ theme ¬∑ layout ¬∑ sleeping-state ¬∑ the frozen-pr snapshot. Stays on `koluSurface`.
- **observed state** ‚Äî the sensors derive it, it is ephemeral and never persisted: cwd ¬∑ git ¬∑ pr ¬∑ agent ¬∑ foreground. Lives on `terminalWorkspaceSurface.awareness`.

**The move: kolu stops *holding* the observation and *reads* it.** The sensors write the `terminalWorkspaceSurface.awareness` collection ‚Äî in-process locally ‚Äî instead of mutating kolu's `terminalMetadata`. kolu *and* the browser read it through **one seam, `awarenessFor(id)`**: the in-process collection today, a host's mirror in R9. `terminalMetadata` shrinks to kolu's authored fields; kolu keeps a few **restore-seeds** (cwd ¬∑ git-cache ¬∑ agent-session) by sampling the observation at save time, so a cold restart re-spawns and re-resolves fast.

```text
before (fold):  sensors ‚îÄ‚îÄ‚îê
                          ‚îú‚îÄ‚îÄ‚ñ∫  terminalMetadata   (one co-owned record + the fence)
                kolu ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò

after (read):   kolu ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚ñ∫  koluSurface.terminalMetadata        (authored)
                sensors ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚ñ∫  terminalWorkspaceSurface.awareness  (observed)
                                kolu reads both via awarenessFor(id) ‚Äî in-process local ¬∑ mirror remote
```

**The fence vanishes ‚Äî it does not move.** The persisted/live fence existed for *one* reason: the ~150 ms agent firehose shared `terminalMetadata` with must-persist fields, so a typed wall (`updateServerLiveMetadata` not arming the autosave) kept the churn off the session save. Once the sensors write the observation surface instead, kolu never persists the live fields at all ‚Äî it subscribes only to `cwd` (human-`cd` frequency) for its restore-seed ‚Äî so the firehose never touches kolu and the **fence has nothing to guard.** One writer per fact: the fold, the co-owned record, *and* the fence all delete. The "irreducible domain logic" the Architecture callout named was a *held copy* that only existed because, before pulam, the observation had nowhere else to live.

It ships **local-only** ‚Äî the in-process surface already exists from R6 ‚Äî so **R8 stands without R9**, and R9 becomes a pure **backing swap** behind `awarenessFor`. The three server-side readers of awareness today (`exportTranscriptHtml`, the active-agent count, the preview's repo-root) move to that same seam ‚Äî the rewiring a *split-the-wire* prototype would owe at R9 anyway, paid once, here.

| | trade-off |
| --- | --- |
| **win ¬∑ one writer per fact** | observation has one home; kolu authors only what it persists. The fold, the co-owned record, and the fence all delete ‚Äî replaced by "compose two surfaces." Adding a sensor field touches the workspace schema + a join-spread, never kolu's wire. |
| **win ¬∑ local == remote** | the same seam answers both; the local/remote axis vanishes behind `awarenessFor` (in-process vs mirror). R9 is a backing swap, not a second data path. |
| **cost ¬∑ the surface moves** | the browser's fs/git + awareness calls move under the `terminalWorkspace` namespace, and server-side readers move to `awarenessFor` ‚Äî a one-time, one-repo change (R6 froze it deliberately; R8 unfreezes). |
| **cost ¬∑ awareness streams in after chrome** | the page paints identity, then live observation lands a beat later (seeded placeholder). *Honest streaming* ‚Äî live presence legitimately follows static identity ‚Äî not a torn-record artifact. A *server restart* shows awareness briefly blank until sensors/mirror catch up; a *browser refresh* is fine (snapshot-then-delta). The warm-vs-fresh trade kaval already took; theme/layout/`location` stay persisted. |

Net (R-compose-1): the awareness compose deletes the fold, the fence, and the co-owned record in one move, and turns R9 into a backing swap. #1510 built and **shipped this green** ‚Äî two collections joined fine-grained client-side, the `awarenessFor` seam (the R9 backing-swap point), a portable `terminalMetadataFold.test.ts` ‚Äî so it's a **reference impl to port**, not a design to re-derive. ‚öÝÔ∏è One latent bug it caught, now filed as [#1529](https://github.com/juspay/kolu/issues/1529): a no-op-upsert collection needs an explicit publish on **every** lifecycle flip ‚Äî wake must re-publish, or a woken terminal stays showing the dormant body.

**R-compose-2 ‚Äî the git-status stream (the gated move).** The *other* half of "one surface" moves the Code tab's git status off `koluSurface` onto `terminalWorkspaceSurface` ‚Äî a read **procedure** (`git.getStatus`) refreshed by a `{seq}` watcher **stream** (`subscribeRepoChange`). **#1510 got this green, so the transport is not the risk** ‚Äî the pulse reaches the browser. The risk that actually cost #1510 months is one layer up: **the file-tree renderer.** `@pierre/trees` swallows the first controller emit after every re-subscribe (and it re-subscribes on layout changes), so a live mutation landing in that window updates the model but **never repaints the DOM** ‚Äî value right, screen stale. master's old git-status stream dodged it only by *fully remounting* the tree; a persistently-mounted tree (the whole point of flicker-free live updates) exposes it. **So R-compose-2 must carry the fix, not just the data:** the vendored **`@pierre/trees` patch** (always bump the revision on every emit) + the client store written via `reconcile` (a no-op on no-change, so no remount-flood) ‚Äî both lifted from #1510 (`useWatchedRead`). That work graduates standalone in [pulam-web](pulam-web.html#r-pulamweb-4) first, whose hermetic test asserts a **DOM repaint of a mounted tree** under a same-shape change, not just a store re-notify. Then R-compose-2 is a backing-swap onto proven electricity. (Design fork, closed: the consumer can be value-bearing *or* the `{seq}`-pulse-plus-requery ‚Äî #1510 proved the pulse shape green; either way the *renderer* is what must repaint.)

### R9 ‚Äî kolu dials remotes

<Phase id="R9" name="kolu dials remotes" status="todo" needs={["R7", "R-compose-2"]} blocks={["R10"]} />

The linchpin ‚Äî it **needs R7** (the total mirror) and is **blocked-on R-compose-2** (the composed surface + its per-terminal dispatch; R-compose-2 proves the browser-ws-stream leg R9 leans on). kolu-server gains the **N-endpoint registry + the ssh driver** ‚Äî the `{kind:"remote",host}` arm, the `(host,id)` join, a persisted `location` ‚Äî reusing the *same* `getHostSession` the spike (R3) built. It dials the host's **kaval** for the PTY (via the shipped `frontDaemonOverStdio`) ‚Üí a remote tile; and dials its **pulam** over a *long-lived* `HostSession`, **mirroring the whole `terminal-workspace` surface** (R7) into one local handle. Because R8 already made that surface the one kolu serves the browser, R9 just **points the remote terminal's dispatch at the mirror** instead of the in-process impl. **No fold to write** (R8 dissolved it), no second data path. The mirror's **streams** ‚Äî awareness `activity` + the fs/git `{seq}` watchers ‚Äî re-serve to the browser over the *same* `websocketLink` + `surfaceClient` leg **R-compose-2** proves (and R9 turns the quiet `activity` stream live on it), so R-compose-2's [pulam-web](pulam-web.html) graduations gate R9, not just R8. `location` is persisted kolu-side, re-validated on reconnect against the remote kaval's live `cwd`. (Paste/upload is a `TerminalEndpoint` concern ‚Äî the same seam, the `PRECONDITION_FAILED` guard.) The result: a remote terminal as capable as a local one ‚Äî PTY, awareness badge, green dot, working Code tab. The shape is the **lead diagram** above.

**Adoption ‚Äî a dial joins, never clobbers.** If the host already runs a kaval it is **adopted** (the same `adoptOrEnsure` spine the local boot uses); a duplicate is impossible by construction (the pid-gate's atomic `link(2)` EEXIST). On dial:

| the remote kaval is‚Ä¶ | kolu does |
| --- | --- |
| absent | provision + spawn fresh (`adoptOrEnsure` ‚Üí ensure) |
| live ¬∑ wire-compatible | **adopt** ‚Äî connect, never kill; its PTYs reconcile in |
| live ¬∑ a build behind | adopt **+** an "update pending" nudge (orthogonal) |
| live ¬∑ contract-skewed | recycle (kill ‚Üí respawn) ‚Äî only on a typed `DaemonContractSkewError` |
| unreachable (bad dial) | bounded retry, then **degraded ‚Äî never killed** |

One kaval per remote host: dialing *joins* its terminals ("the prod terminals"), never racing to kill them.
