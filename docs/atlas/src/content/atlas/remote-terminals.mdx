## User-facing description

<RemoteCanvas />

You switch hosts from the **ChromeBar** ‚Äî *kinda like tmux sessions* ‚Äî and a remote host's terminals sit on the canvas **beside** your local ones. A remote tile wears an `ssh` badge and carries the *same* live awareness a local tile does ‚Äî git branch/dirty, PR status, agent state, and a **green dot** when it's moving bytes ‚Äî and it survives a network blip or a kolu-server deploy with full scrollback. One local host is always present; remotes are added on demand, auto-detected from your ssh config.

## Architecture-level changes

<Svg svg={arch} caption="The multi-host shape. Every host runs the same hashed kaval daemon; kolu-server keys its terminal endpoint by hostId behind one TerminalEndpoint seam. The local driver (R2, shipped) spawns + supervises a kaval over a unix socket; the ssh driver (R9) reaches + provisions the same kaval closure over ssh stdio; the remote mirror (R9) reaches a host-side pulam's awareness (kolu serves the shared awareness collection raw as of R8, joined with the authored half at the reader) + fs/git (read in R9). Local awareness runs in-process; only remote crosses a process boundary." />

**One endpoint, bound to a dialed kaval.** A kaval is a daemon you *dial* ‚Äî local over a unix socket, remote over ssh ‚Äî so remote is **not a second backend**. There is *one* `TerminalEndpoint` per terminal, resolved by `HostLocation` (`{kind:"local"}` today, `{kind:"remote",hostId}` in R9). That resolver is meant to be the **sole** place a tile maps to its kaval; everything downstream talks to the backend and never asks "which kind?". No `RemoteTerminalEndpoint` ‚Äî but the resolver doesn't exist yet: today `localTerminalEndpoint` is imported directly, so **R9.1 first routes those imports through the resolver** (a small retrofit), then **R9.2 adds the ssh *driver*** as an additive sibling of R2's local driver.

Two volatility axes stay open (R1/R2 closed the rest):

| Axis ‚Äî changes for its own reason | Encapsulated by |
| --- | --- |
| Where a terminal's state lives ‚Äî this machine vs an ssh host | the `HostLocation`-resolved kaval endpoint ‚Äî one impl + a discriminator, not a second backend |
| How the backend reaches its agent ‚Äî transport ¬∑ framing ¬∑ reconnect | `HostSession` + `@kolu/surface/links/stdio` ‚Äî unix socket today, ssh in R9 |

**Zero domain knowledge crosses the transport** (Hickey's cut): the agent runs the unmodified sensors and the boundary ships their *values*. An earlier per-domain design (`RemoteGitInfoProvider`, ‚Ä¶) was transport adapters wearing domain costumes; the single-endpoint seam dissolved them.

<Callout kind="note" title="kolu no longer keeps its own copy of awareness ‚Äî R8 shipped">
kolu used to fuse awareness into its **own** `terminalMetadata` record ‚Äî a private copy. **R8** (<PrLink pr={1594} />) split it: a single-writer awareness store is served **raw** on `terminalWorkspaceSurface.awareness`, kolu keeps only what it *authors* (`kolu.authored`), and the **client joins the two at the reader** ‚Äî no server-side fusion. Detail in **R8** and [the terminal model](terminal-metadata-model.html).
</Callout>

**Already paid for, so R5‚ÄìR10 are narrow.** `@kolu/surface-nix-host` ships `HostSession`, `provisionAgent` (ships the *derivation* ‚Äî a darwin parent drives a linux remote with no cross-builder), and `resolveSystem`; `@kolu/surface` ships `mirrorRemoteSurface` (now a total dual). R2 already shipped the host-count-agnostic spine: `@kolu/surface-daemon-supervisor` and the per-connect `system.version` handshake. So R9 is an ssh driver behind shipped seams; R8 made kolu serve the shared awareness (<PrLink pr={1594} />); R7 *completed* the surface mirror (<PrLink pr={1505} />); R6 grew pulam's fs/git (<PrLink pr={1506} />); R5 is one wire signal.

### R8 ‚Äî kolu serves awareness raw; the client joins it ‚Äî *shipped* <PrLink pr={1594} />

<Phase id="R8" name="kolu serves awareness raw; client joins authored ‚ãà awareness" status="shipped" needs={["R6"]} blocks={["R9", "R9a"]} links={[{ label: "#1594", href: "https://github.com/juspay/kolu/pull/1594" }]} />

**Shipped** (<PrLink pr={1594} />). kolu used to fuse awareness into its own private `terminalMetadata` record. R8 bisected it: the eight sensor fields live in a **single-writer awareness store** served **raw** on `terminalWorkspaceSurface.awareness` (`AwarenessValue`); kolu keeps only what it *authors* on `kolu.authored` (`AuthoredTerminal` ‚Äî location ¬∑ chrome ¬∑ the active|sleeping discriminant). **Neither side fuses them** ‚Äî `surfaceCtx.collections.terminalMetadata` is a compile error. The **client joins the two halves at the reader** (`useTerminalMetadata` ‚Üí `composeTerminalMetadata`), and the same join authors the on-disk `SavedTerminal`, so disk and the read can't diverge. One writer per fact, and the bisection reaches the consumer. Full model: [the terminal model](terminal-metadata-model.html).

**One surface, two homes.** kolu-server and the `pulam` daemon both serve `terminalWorkspaceSurface`, but assemble it via **one factory** ‚Äî `@kolu/terminal-workspace/serveTerminalWorkspace`, the volatility-boundary twin of `serveFsGit`: the `version` cell + fs/git procedures/streams live there once, and each home injects only its **awareness backing** (kolu projects off its registry; pulam reads its own store) and its **`activity` source** (quiet for kolu, live for pulam). So R8 **unblocks [R9a](#r9a)** ‚Äî kolu now serves an awareness a second process can read ‚Äî and makes remote awareness in R9 a **backing injection**, not a rewrite.

**fs/git is not part of R8 ‚Äî it rides R9.** `terminalWorkspaceSurface` deliberately serves fs/git as **procedures + pulse**, not value-bearing streams: `fs.listAll`/`git.getStatus` are request‚Üíresponse procedures, and `subscribeRepoChange`/`subscribeFileChange` are payload-free `{seq}` pulses you re-query on (`surface.ts:89-109` ‚Äî *"re-queries procedures rather than streaming full diffs over the wire"*). kolu's Code tab today reads `koluSurface`'s **value-bearing** streams (`app.streams.gitStatus/fsListAll/gitDiff.use`, `CodeTab.tsx:314-373`). Making kolu read the shared surface's fs/git means **rewriting the Code tab from value-bearing streams to procedure + pulse-then-requery** ‚Äî a large client change the surface ties to mirroring the surface whole. So it lands with **R9**, where kolu mirrors the whole surface anyway.
