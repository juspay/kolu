## User-facing description

<RemoteCanvas />

You switch hosts from the **ChromeBar** — *kinda like tmux sessions* — and a remote host's terminals sit on the canvas **beside** your local ones. A remote tile wears an `ssh` badge and carries the *same* live awareness a local tile does — git branch/dirty, PR status, agent state, and a **green dot** when it's moving bytes — and it survives a network blip or a kolu-server deploy with full scrollback. One local host is always present; remotes are added on demand, auto-detected from your ssh config.

## Architecture-level changes

<Svg svg={arch} caption="The multi-host shape. Every host runs the same hashed kaval daemon; kolu-server keys its terminal endpoint by hostId behind one TerminalEndpoint seam. The local driver (R2, shipped) spawns + supervises a kaval over a unix socket; the ssh driver (R9) reaches + provisions the same kaval closure over ssh stdio; the remote mirror (R9) reaches a host-side pulam's awareness + fs/git, composed into one surface (R8). Local awareness runs in-process; only remote crosses a process boundary." />

**One endpoint, bound to a dialed kaval.** A kaval is a daemon you *dial* — local over a unix socket, remote over ssh — so remote is **not a second backend**. There is *one* `TerminalEndpoint`, bound to a `HostLocation`-resolved kaval endpoint (`{kind:"local"}` today, `{kind:"remote",host}` in R9). That resolver is the **sole** place a tile maps to its kaval; everything downstream talks to the backend and never asks "which kind?". No `RemoteTerminalEndpoint` — R9 adds the ssh *driver* as an additive sibling of R2's local driver, never a retrofit.

Two volatility axes stay open (R1/R2 closed the rest):

| Axis — changes for its own reason | Encapsulated by |
| --- | --- |
| Where a terminal's state lives — this machine vs an ssh host | the `HostLocation`-resolved kaval endpoint — one impl + a discriminator, not a second backend |
| How the backend reaches its agent — transport · framing · reconnect | `HostSession` + `@kolu/surface/links/stdio` — unix socket today, ssh in R9 |

**Zero domain knowledge crosses the transport** (Hickey's cut): the agent runs the unmodified sensors and the boundary ships their *values*. An earlier per-domain design (`RemoteGitInfoProvider`, …) was transport adapters wearing domain costumes; the single-endpoint seam dissolved them.

<Callout kind="note" title="The awareness 'fold' was never domain logic — R8 deletes it">
Through R7, remote awareness was *folded* server-side into the **co-owned** `terminalMetadata` record, split across a persisted/live **fence** (`pr·agent·foreground` live; `cwd·git·…` persisted) whose only job was keeping the ~150 ms agent firehose off the session autosave. Both were artifacts of *one co-owned record*. R8 gives observation its own home: the sensors write `terminalWorkspaceSurface.awareness`, and kolu (and the browser) **read** it through one seam (`awarenessFor`) instead of folding a copy in. One writer per fact — so the fold, the co-owned record, *and* the fence all **vanish** (the firehose never touches kolu). Detail + trade-offs in **R8**.
</Callout>

**Already paid for, so R5–R10 are narrow.** `@kolu/surface-nix-host` ships `HostSession`, `provisionAgent` (ships the *derivation* — a darwin parent drives a linux remote with no cross-builder), and `resolveSystem`; `@kolu/surface` ships `mirrorRemoteSurface` (now a total dual). R2 already shipped the host-count-agnostic spine: `@kolu/surface-daemon-supervisor` and the per-connect `system.version` handshake. So R9 is an ssh driver behind shipped seams; R8 composes one surface (the fold dissolves); R7 *completed* the surface mirror (<PrLink pr={1505} />); R6 grew pulam's fs/git (<PrLink pr={1506} />); R5 is one wire signal.

### R5 — overflow-recovery loop

~130 LoC w/ tests; the next step. kaval sheds a slow attach subscriber by *ending* its iterator, today indistinguishable on the wire from a PTY exit, so the client treats the drop as terminal and freezes scrollback (`onOverflow` exists but is unwired). Thread a typed *overflow → re-attach* signal through the attach contract so the client re-attaches for a **fresh snapshot** (reusing the existing `onRetry` reset), distinct from PTY-exit. **Ships independently of the remote work**: a slow subscriber is reproducible on the *local* socket, so it is CI-testable today, and the fix hardens the attach contract for *both* transports — a clean prerequisite for the remote reconnect R8 leans on. [herdr vs kolu](herdr-vs-kolu.html) flagged the same gap.

<a id="r6"></a>

### R6 — `@kolu/terminal-workspace`: one library, one fs/git impl, two homes

**Shipped** — <PrLink pr={1506} />. The host-side library that knows a terminal's working directory becomes **`@kolu/terminal-workspace`** — the shipped `@kolu/terminal-awareness` (R4.1) grown to cover fs/git too, run *in-process* locally (kolu) and hosted by *pulam* remotely: the same two homes the sensors already proved. R6 ships **one fs/git impl** (`createTerminalWorkspaceEndpoint`), *not* one surface both homes already serve — the two homes re-expose that one impl through two **deliberately different** contract shapes:

- `awareness` (collection) + `activity` (stream) — the derived status badge + the green-dot liveness, served on `terminalWorkspaceSurface`. **Shipped.**
- the Code tab's fs/git reads (`listAll` · `readFile` · `getStatus` · `getDiff` + repo/file change watching) — **new in R6**, lifted from kolu-server's `localEndpoint` (a thin wrapper over `kolu-git` + `serve-dir`) into the shared library. kolu-server (in-process) re-exposes them on `koluSurface`'s **value-bearing streams**; pulam (remote) serves them on `terminalWorkspaceSurface` as **procedures + `{seq}` pulse watcher streams** a consumer requeries on.

This kills the duplication wart: without it, pulam would **reimplement** fs/git and kolu would carry *two* fs/git impls (its own local, pulam's remote). Instead there is **one** — in-process locally, via pulam remotely. Only the thin fs/git wrapper moves; the terminal-endpoint *orchestration* around it (spawn · adopt · the registry) stays kolu's and gets a remote impl in R9. The single shared **surface** both homes serve is *not* yet true — it arrives in **R8**, which composes `terminalWorkspaceSurface` into the one surface kolu serves the browser (the server-side fold dissolving with it); **R9** then dials remotes and points that surface's per-terminal dispatch at the mirror. Until then the two contract shapes are kept deliberately distinct. Nothing is folded into `AwarenessValue` — procedures were never part of it.

<a id="r7"></a>

### R7 — `mirrorRemoteSurface` is now a total dual — *shipped* <PrLink pr={1505} />

`mirrorRemoteSurface` is the consume-side **dual of `implementSurface`** — and a dual must mirror *every* primitive its producer wires. `implementSurface` wires procedures too, but the mirror's `SurfaceSink` carried only `cells`/`collections`/`streams`/`events`: a **leaky half-mirror**, harmless only because every surface mirrored so far had **zero procedures**. R7 closed the gap by kind: **streaming state stays PUSH** (subscribed and folded into the sink); a **procedure is PULL** (a local call runs on the remote and returns), so it can't sit in the push-only sink — it comes back as a **forwarding stub**. The return is therefore `{ procedures, done }`: `procedures.<ns>.<verb>(input)` relays to the remote (total — every procedure gets a stub, since a passthrough has no standing cost), and `done` is the former settle-on-link-close promise. Graft the stubs into a re-serve and `serve ∘ mirror ≈ identity` — the location-transparency the whole epic rests on — **pinned in-repo** by a hermetic unit test (`implementSurface` → `directLink` → mirror → re-serve → call). A `@kolu/surface` change → the **drishti merge-gate**.

**Proven by drishti — the canonical external second consumer.** [drishti](https://github.com/srid/drishti) is the browser ↔ Bun ↔ ssh process-monitor that already mirrors per-host *process* surfaces (the graduation that first proved `@kolu/surface` is electricity), and it was **read-only**. R7's external proof is drishti gaining a **"Kill process"** action — the first *forwarded procedure* on a mirrored surface, exercised end-to-end (browser → Bun → ssh → remote agent → `kill(pid)`), **zero kolu**. That proof is the **merge-gate** for this kolu PR, not optional decoration: the return shape changed from `Promise<void>` to `{ procedures, done }`, and drishti `master` still `await`s the old shape on its streaming sinks ([drishti #70](https://github.com/srid/drishti/pull/70) adopted the #1497 `mirrorRemoteSurface`, all CI green). So the paired [drishti R7 PR #71](https://github.com/srid/drishti/pull/71) — audit that `await` to `.done`, pin drishti's kolu source to this R7 revision, go green — is now **open and gates #1505**: its full CI is green against the new `{ procedures, done }` API — all 18 odu nodes (every lane on both `aarch64-darwin` and `x86_64-linux`), the "Kill process" forwarded-procedure action landing on top of the migrated mirror — exactly the lane set #70 passed to gate #1497. Because that proof lives entirely in drishti's own surface, **R7 stands alone — it needs neither R6 nor kolu**; R9 then consumes a *proven* total mirror for the remote dial. **One caution this plan earlier blurred:** the mirror (remote → local, proven here by drishti) is a *different primitive* from **kolu composing-and-serving a stream to the browser** over `websocketLink` + `surfaceClient`. That second leg is **R8b's**, and it is proven separately by **[pulam-web](pulam-web.html)** (R4.8) — not by this mirror. drishti proved the mirror; *pulam-web* proves the browser leg.

<a id="r8"></a>

### R8 — compose terminal-workspace into kolu-server (the fold vanishes)

R6 left **one impl, two contract shapes**: kolu re-exposes fs/git on `koluSurface`'s own streams (in-process), pulam serves `terminalWorkspaceSurface` (remote). R8 collapses that to **one surface** — and settles the question the epic kept reopening, *who owns awareness?* The answer is **not kolu**.

**Two moves, very different risk** — a throwaway prototype (<PrLink pr={1510} />, *discarded in flight*) proved they must be split. Neither half is started as merged code; the prototype is kept only as the lesson. **R8a — awareness:** the one-writer-per-fact compose below. The prototype ran it end-to-end with *zero* failures, so the design is **proven** — R8a is a **clean build on settled ground**, not an open question. **R8b — live git-status stream:** moving the Code tab's git status onto `terminalWorkspaceSurface`'s `{seq}` watcher **stream** over `websocketLink` + `surfaceClient` — a primitive **no phase ever graduated**, and exactly where the prototype stuck. R8b is **gated on [pulam](pulam.html#r46)'s R4.7 + R4.8**, which prove that leg standalone first (the drishti discipline). The architecture below is **R8a**; R8b is its own subsection at the end.

A terminal is two things that were never one record:

- **authored state** — kolu writes it, it is durable: id · restore-cwd · theme · layout · sleeping-state · the frozen-pr snapshot. Stays on `koluSurface`.
- **observed state** — the sensors derive it, it is ephemeral and never persisted: cwd · git · pr · agent · foreground. Lives on `terminalWorkspaceSurface.awareness`.

**The move: kolu stops *holding* the observation and *reads* it.** The sensors write the `terminalWorkspaceSurface.awareness` collection — in-process locally — instead of mutating kolu's `terminalMetadata`. kolu *and* the browser read it through **one seam, `awarenessFor(id)`**: the in-process collection today, a host's mirror in R9. `terminalMetadata` shrinks to kolu's authored fields; kolu keeps a few **restore-seeds** (cwd · git-cache · agent-session) by sampling the observation at save time, so a cold restart re-spawns and re-resolves fast.

```text
before (fold):  sensors ──┐
                          ├──►  terminalMetadata   (one co-owned record + the fence)
                kolu ─────┘

after (read):   kolu ────────►  koluSurface.terminalMetadata        (authored)
                sensors ─────►  terminalWorkspaceSurface.awareness  (observed)
                                kolu reads both via awarenessFor(id) — in-process local · mirror remote
```

**The fence vanishes — it does not move.** The persisted/live fence existed for *one* reason: the ~150 ms agent firehose shared `terminalMetadata` with must-persist fields, so a typed wall (`updateServerLiveMetadata` not arming the autosave) kept the churn off the session save. Once the sensors write the observation surface instead, kolu never persists the live fields at all — it subscribes only to `cwd` (human-`cd` frequency) for its restore-seed — so the firehose never touches kolu and the **fence has nothing to guard.** One writer per fact: the fold, the co-owned record, *and* the fence all delete. The "irreducible domain logic" the Architecture callout named was a *held copy* that only existed because, before pulam, the observation had nowhere else to live.

It ships **local-only** — the in-process surface already exists from R6 — so **R8 stands without R9**, and R9 becomes a pure **backing swap** behind `awarenessFor`. The three server-side readers of awareness today (`exportTranscriptHtml`, the active-agent count, the preview's repo-root) move to that same seam — the rewiring a *split-the-wire* prototype would owe at R9 anyway, paid once, here.

| | trade-off |
| --- | --- |
| **win · one writer per fact** | observation has one home; kolu authors only what it persists. The fold, the co-owned record, and the fence all delete — replaced by "compose two surfaces." Adding a sensor field touches the workspace schema + a join-spread, never kolu's wire. |
| **win · local == remote** | the same seam answers both; the local/remote axis vanishes behind `awarenessFor` (in-process vs mirror). R9 is a backing swap, not a second data path. |
| **cost · the surface moves** | the browser's fs/git + awareness calls move under the `terminalWorkspace` namespace, and server-side readers move to `awarenessFor` — a one-time, one-repo change (R6 froze it deliberately; R8 unfreezes). |
| **cost · awareness streams in after chrome** | the page paints identity, then live observation lands a beat later (seeded placeholder). *Honest streaming* — live presence legitimately follows static identity — not a torn-record artifact. A *server restart* shows awareness briefly blank until sensors/mirror catch up; a *browser refresh* is fine (snapshot-then-delta). The warm-vs-fresh trade kaval already took; theme/layout/`location` stay persisted. |

Net (R8a): the awareness compose deletes the fold, the fence, and the co-owned record in one move, and turns R9 into a backing swap. The discarded prototype validated exactly this end-to-end (zero awareness failures); the clean build keeps the win and drops that prototype's split-then-rejoin scaffolding.

**R8b — the live git-status stream (the gated move).** The *other* half of "one surface" moves the Code tab's git status off `koluSurface` onto `terminalWorkspaceSurface` — served as a read **procedure** (`git.getStatus`) refreshed by a per-subscription **`{seq}` watcher stream** (`subscribeRepoChange`). The procedure crosses the ws fine; the **stream** is the catch. It is the **first time anything drives this surface's `source`-arm streams over `websocketLink` → `surfaceClient` into the browser** — R6 only exercised the `awareness` *collection*; R7/drishti proved *forwarded procedures* and drishti's *own* stream over drishti's stack, never kolu's browser. The prototype stuck there, and blind (the prod build hides the console; the server log is in an ephemeral sandbox). So R8b is **gated on [pulam](pulam.html#r46) R4.7 + R4.8**: R4.7 proves the `{seq}` *shape* (a live **git-status** view in `pulam-tui` — working tree + branch, no file content; the pulse-plus-requery loop *is* the proof), R4.8 proves kolu's exact *ws + `surfaceClient` + reconcile* leg ([pulam-web](pulam-web.html), the same view in the browser) — each verified by raw capture over the **real** transport, *never directLink*. Only then does R8b compose git status, a backing-swap onto twice-proven electricity — and the **same** pulse-plus-requery then carries the rest of the Code tab (file diffs/content via `subscribeFileChange`), identical mechanism, no new risk. (**The recipe — resolved by [pulam-web](pulam-web.html) Step 0** ([drishti #72](https://github.com/srid/drishti/pull/72)): `gitStatus` is genuinely *value-bearing* (each frame is the full status), so wrap it server-side and consume via `.streams.use()` **read fine-grained** — read the cell's fields off the subscription, never coarse-read-and-copy, which silently coalesces same-shape frames and freezes the value. Guard it with a same-shape-delta regression test. The design fork is closed, not a guess.)

<a id="r9"></a>

### R9 — kolu dials remotes

The linchpin — it **needs R7** (the total mirror) and is **blocked-on R8b** (the composed surface + its per-terminal dispatch; R8b proves the browser-ws-stream leg R9 leans on). kolu-server gains the **N-endpoint registry + the ssh driver** — the `{kind:"remote",host}` arm, the `(host,id)` join, a persisted `location` — reusing the *same* `getHostSession` the spike (R3) built. It dials the host's **kaval** for the PTY (via the shipped `frontDaemonOverStdio`) → a remote tile; and dials its **pulam** over a *long-lived* `HostSession`, **mirroring the whole `terminal-workspace` surface** (R7) into one local handle. Because R8 already made that surface the one kolu serves the browser, R9 just **points the remote terminal's dispatch at the mirror** instead of the in-process impl. **No fold to write** (R8 dissolved it), no second data path. The mirror's **streams** — awareness `activity` + the fs/git `{seq}` watchers — re-serve to the browser over the *same* `websocketLink` + `surfaceClient` leg **R8b** proves (and R9 turns the quiet `activity` stream live on it), so R8b's [pulam](pulam.html#r46) graduations gate R9, not just R8. `location` is persisted kolu-side, re-validated on reconnect against the remote kaval's live `cwd`. (Paste/upload is a `TerminalEndpoint` concern — the same seam, the `PRECONDITION_FAILED` guard.) The result: a remote terminal as capable as a local one — PTY, awareness badge, green dot, working Code tab. The shape is the **lead diagram** above.

**Adoption — a dial joins, never clobbers.** If the host already runs a kaval it is **adopted** (the same `adoptOrEnsure` spine the local boot uses); a duplicate is impossible by construction (the pid-gate's atomic `link(2)` EEXIST). On dial:

| the remote kaval is… | kolu does |
| --- | --- |
| absent | provision + spawn fresh (`adoptOrEnsure` → ensure) |
| live · wire-compatible | **adopt** — connect, never kill; its PTYs reconcile in |
| live · a build behind | adopt **+** an "update pending" nudge (orthogonal) |
| live · contract-skewed | recycle (kill → respawn) — only on a typed `DaemonContractSkewError` |
| unreachable (bad dial) | bounded retry, then **degraded — never killed** |

One kaval per remote host: dialing *joins* its terminals ("the prod terminals"), never racing to kill them.
