**The last leg of <Issue n={951} /> — kolu dials remote hosts.** R8 (<PrLink pr={1594} />) shipped the right **reader-join** (a terminal is `kolu.authored ⋈ terminalWorkspace.awareness`, joined at the reader) — *keep that*. The question this note settles is how kolu gets the **awareness** half for a *remote* host, and that architecture is now **decided**: split **observing** from **remembering**. A **memoryless producer** runs beside each host's kaval and emits `Observation` events; **kolu alone folds** them, owns the memory (on `kolu.authored`), and stamps recency with **its own clock**. The full types/API/flows are in **[the awareness design](awareness-derive-store.html)**; this note is the phase plan of record. The parent [remote-terminals](remote-terminals.html) keeps the one-line rows; the model is in [the terminal model](terminal-metadata-model.html).

<Callout kind="note" title="The settled design — and the two directions it replaced">
The awareness boundary is **observe vs. remember** ([the awareness design](awareness-derive-store.html), outcome of a grounded 3-agent debate). It supersedes two earlier attempts, both now dead: (1) the **local-pulam-PROCESS** (#1614, **CLOSED**) — it conflated a process boundary with code decoupling and introduced a fold-clobber intrinsic to mirroring an ephemeral process; (2) **createPulam / "share the assembly"** — it shared the *wrong thing* (the assembly still fused deriving with storing, so the foot-gun and clobber rode along to every consumer). The converged fix doesn't share the assembly; it **deletes** it: the producer is memoryless (it can't even spell memory, by type), so there is nothing to clobber and nothing to assemble. **R9·lib therefore dissolves** (below).
</Callout>

### S1 · S2 — the awareness foundation, and why R9·lib dissolves

<Phase id="S1" name="memoryless awareness producer — observations, no seed, Observation only" status="done" needs={["R6", "R4"]} />
<Phase id="S2" name="kolu's fold owns memory — kolu's clock, memory→authored" status="done" needs={["S1"]} />

> **Converged + landed.** The S1/S2 model below was simplified across a long review and shipped by the [awareness-derive-store cutover](awareness-derive-store.html) — read that note for the final types. The summary here is updated to match: the three-category `{cache, live, memory}` patch model and the per-event `fromSnapshot` flag were **dropped** (snapshot-vs-delta rides the frame, not the event).

**S1** rewrites the sensors into a **memoryless producer**: it emits per-field `AwarenessObservation` events, holds no durable memory, takes no seed, and its export type `Observation` (the five re-observable fields) **cannot spell** the memory fields. It *deletes* the `AwarenessSink`, both `makeAwarenessSink` impls, the `AwarenessRecord`, and the apply-and-publish read-back contract. **S2** gives **kolu** the only fold: `fold(cur, o, ctx) → KoluAwareness` (`{observed, memory}`), last-write-wins for the five observed fields, stamping recency with kolu's clock on a live agent-identity change and remembering `lastAgentCommand` (the resume target rides the authored record as the fold-derived discriminated `restoreTarget`). It writes memory onto `kolu.authored` — so every awareness collection everywhere is `Observation`.

**Why R9·lib dissolves:** createPulam was an attempt to *share the assembly* the daemon and kolu each hand-rolled. Under S1+S2 there is **no assembly** — the producer and the fold are pure leaves, each home's loop is ~8 lines, and the producer can't write memory, so there is nothing to clobber and nothing to consolidate. The `R9·lib` / createPulam phase is **removed**.

- **Needs:** R6 (the sensor library) and R4 (the pulam daemon) — both shipped.
- **Done (S1):** the producer emits observations and touches no host store; `AwarenessSink`/`makeAwarenessSink`×2/`AwarenessRecord`/seed are gone. **Done (S2):** kolu's fold is the sole writer of memory (onto `kolu.authored`); the write-fence, clock authority, and single-memory-writer are enforced **by type** (tests in the awareness note). The remote `awarenessEvents` *wire* stream is R9.3; R9.0 folds the in-process producer and pulam serves an `Observation` collection.

### R9.0 — kolu runs the producer in-process, folds

<Phase id="R9.0" name="kolu runs the memoryless producer in-process and folds" status="todo" needs={["S2"]} />

kolu's local awareness is the **memoryless producer called in-process** (no daemon, no socket, no mirror), its observations fed straight into kolu's fold. kolu seeds `current` from its durable record, folds with its clock, writes the memory fields to `kolu.authored` and the rest to the `awareness` collection. The reader-join is unchanged (`composeTerminalMetadata` now reads memory from the authored half). Clobber-free **by type** (the producer can't spell memory).

- **Migration:** a **hard cutover**, no flag/fallback; plus the memory→`kolu.authored` schema move (`SCHEMA_VERSION` bump + state migration).
- **Needs:** S2.
- **Done:** kolu runs no bespoke sink/assembly; local awareness is the producer + kolu's fold; the reader-join and disk persistence hold; a fold differential test passes. Detail: [the awareness design](awareness-derive-store.html).

### R9.2 — ssh kaval driver → a remote PTY tile

<Phase id="R9.2" name="ssh kaval driver — remote PTY" status="todo" needs={["R9.1"]} />

### R9.3 — remote awareness = subscribe the remote event stream + the same fold

<Phase id="R9.3" name="remote awareness — subscribe the remote producer's event stream" status="todo" needs={["S2", "R9.2", "R7"]} />

A remote host runs the **same memoryless producer** (in its pulam daemon), serving an ordered `awarenessEvents` stream of `Observation`. kolu subscribes over a long-lived `getHostSession` (mirrored via R7's total dual) and folds those events through the **same** fold it uses locally — kolu seeds `current` from its durable record, stamps recency with **its** clock, writes memory to `kolu.authored`. Because kolu seeds from memory and the stream is snapshot-then-deltas, the **snapshot frame is the reconcile** — there is no separate `reconcileRemoteSnapshot`. Only the transport differs (in-process vs ssh); the fold, the reader-join, and disk persistence are identical.

- **Needs:** S2 (the producer + fold), R9.2 (the remote PTY), R7 (the total dual).
- **Done:** a remote tile shows live awareness folded from the remote producer's stream, via the **same** fold as local; a differential test pins it equal to the producer's observed state. Detail: [the awareness design](awareness-derive-store.html).

### R9.4 — persisted location, reconnect, adoption

<Phase id="R9.4" name="persisted location + reconnect + adoption" status="todo" needs={["R9.2"]} />

Harden the remote lifecycle: re-validate the persisted `location` on reconnect, and cover the adoption/reconcile cases.

| the remote kaval is… | kolu does |
| --- | --- |
| absent | provision + spawn fresh (`adoptOrEnsure` → ensure) |
| live · wire-compatible | **adopt** — connect, never kill; its PTYs reconcile in |
| live · a build behind | adopt **+** an "update pending" nudge |
| live · contract-skewed | recycle (kill → respawn) — only on a typed `DaemonContractSkewError` |
| unreachable | bounded retry, then **degraded — never killed** |

- **Open (decide here):** *what* is re-validated on reconnect and the fail behavior. `location` is the host discriminator (`kolu.authored`); `cwd` is an observed field — different homes. Spell out the comparison and the fail-fast (crash loudly vs re-derive); don't invent it at implementation time.
- **Needs:** R9.2.
- **Done:** a kolu-server restart re-adopts remote PTYs; each adoption-table row has a test.

### R9.5 — the Code-tab fs/git rewrite (procedure + pulse)

<Phase id="R9.5" name="Code-tab fs/git → procedure + pulse" status="todo" needs={["R-pulamweb-4", "R9.3"]} />
