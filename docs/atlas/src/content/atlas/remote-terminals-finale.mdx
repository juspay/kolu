**The last leg of <Issue n={951} /> — kolu dials remote hosts.** R8 (<PrLink pr={1594} />) shipped the right **reader-join** (a terminal is `kolu.authored ⋈ terminalWorkspace.snapshots`, joined at the reader) — *keep that*. The question this note settles is how kolu gets the **awareness** half for a *remote* host, and that architecture is now **decided**: split **observing** from **remembering**. A **memoryless producer** runs beside each host's kaval and emits `TerminalSnapshot` events; **kolu alone folds** them, owns the memory (on `kolu.authored`), and stamps recency with **its own clock**. The full types/API/flows are in **[the awareness design](awareness-derive-store.html)**; this note is the phase plan of record. The parent [remote-terminals](remote-terminals.html) keeps the one-line rows; the model is in [the terminal model](terminal-metadata-model.html).

<Callout kind="note" title="The settled design — and the two directions it replaced">
The awareness boundary is **observe vs. remember** ([the awareness design](awareness-derive-store.html), outcome of a grounded 3-agent debate). It supersedes two earlier attempts, both now dead: (1) the **local-pulam-PROCESS** (#1614, **CLOSED**) — it conflated a process boundary with code decoupling and introduced a fold-clobber intrinsic to mirroring an ephemeral process; (2) **createPulam / "share the assembly"** — it shared the *wrong thing* (the assembly still fused deriving with storing, so the foot-gun and clobber rode along to every consumer). The converged fix doesn't share the assembly; it **deletes** it: the producer is memoryless (it can't even spell memory, by type), so there is nothing to clobber and nothing to assemble. **R9·lib therefore dissolves** (below).
</Callout>

### S1 · S2 — the awareness foundation, and why R9·lib dissolves

<Phase id="S1" name="memoryless awareness producer — observations, no seed, TerminalSnapshot only" status="done" needs={["R6", "R4"]} />
<Phase id="S2" name="kolu's fold owns memory — kolu's clock, memory→authored" status="done" needs={["S1"]} />

> **Converged + landed.** The S1/S2 model below was simplified across a long review and shipped by the [awareness-derive-store cutover](awareness-derive-store.html) — read that note for the final types. The summary here is updated to match: the three-category `{cache, live, memory}` patch model and the per-event `fromSnapshot` flag were **dropped** (snapshot-vs-delta rides the frame, not the event).

**S1** rewrites the sensors into a **memoryless producer**: it emits per-field `TerminalEvent` events, holds no durable memory, takes no seed, and its export type `TerminalSnapshot` (the five re-observable fields) **cannot spell** the memory fields. It *deletes* the `AwarenessSink`, both `makeAwarenessSink` impls, the `AwarenessRecord`, and the apply-and-publish read-back contract. **S2** gives **kolu** the only fold: `fold(cur, o, ctx) → TerminalState` (`{snapshot, memory}`), last-write-wins for the five snapshot fields, stamping recency with kolu's clock on a live agent-identity change and remembering `lastAgentCommand` (the resume target rides the authored record as the fold-derived discriminated `restoreTarget`). It writes memory onto `kolu.authored` — so every snapshots collection everywhere is `TerminalSnapshot`.

**Why R9·lib dissolves:** createPulam was an attempt to *share the assembly* the daemon and kolu each hand-rolled. Under S1+S2 there is **no assembly** — the producer and the fold are pure leaves, each home's loop is ~8 lines, and the producer can't write memory, so there is nothing to clobber and nothing to consolidate. The `R9·lib` / createPulam phase is **removed**.

- **Needs:** R6 (the sensor library) and R4 (the pulam daemon) — both shipped.
- **Done (S1):** the producer emits observations and touches no host store; `AwarenessSink`/`makeAwarenessSink`×2/`AwarenessRecord`/seed are gone. **Done (S2):** kolu's fold is the sole writer of memory (onto `kolu.authored`); the write-fence, clock authority, and single-memory-writer are enforced **by type** (tests in the awareness note). The remote `terminalEvents` *wire* stream is R9.3; R9.0 folds the in-process producer and pulam serves a `TerminalSnapshot` collection.

## R9 re-sequenced — prep-first, F-REMOTE complete-everything

The original `R9.2 → R9.3 → R9.4 → R9.5` order shipped a remote tile **in pieces** (PTY first; Code tab, restore, awareness bolted on later) — which violates the **hard constraint** settled across the `debates/r92-r93/` rounds and ratified by an adversarial-verification workflow: *no PR may break existing functionality, and a remote tile must be **as complete as a local tile** the moment a user can create one — nothing deferred.* So R9 re-sequences into **three parallel, local-preserving prep PRs** (each makes one *concern* host-aware as a no-op on local) → **F-REMOTE** (the one PR that lights up a complete remote tile) → **R10** (host-picker UX). R9.2/R9.3/R9.4 dissolve into this shape; R9.5 becomes prep (PR-2); R-pulamweb-4 leaves kolu's critical path (kolu mirrors the remote pulam server-side via R7, not the pulam-web browser leg).

<Callout kind="note" title="Why three prep PRs, not six (decompose by concern, not by seam)">
`HostLocation` is already the source of truth for "which host" (R9.1) — so completing it is **one idea per concern** (the *lifecycle*, *fs/git + file-IO*, *awareness*), not one PR per file touched. Each prep PR is a pure local no-op; all three are **logically independent → fully parallel** — PR-2 *reads* the already-persisted `entry.meta.location`, it does not need PR-1 to *set* it. The only coordination is a surface-version rebase between PR-2 and PR-3 (both bump `TERMINAL_WORKSPACE_CONTRACT_VERSION`). The adversarial pass also recovered **three first-class capabilities silently pinned to the local host** — sleep/wake, paste/upload, transcript export — now first-class in the prep + F-REMOTE (each routed to its host, never silently wrong-host).
</Callout>

### F-REMOTE — the complete remote tile *(the one user-visible feature; nothing deferred)*

<Phase id="F-REMOTE" name="remote terminal — PTY · fs/git · full awareness · restore · sleep/wake · paste/upload · transcript" status="todo" needs={["PR-1", "PR-2", "PR-3", "R7"]} />

- **Session restore incl. boot re-dial-re-adopt** — a *new* F-REMOTE loop reads saved remote `hostId`s, dials each host's kaval **and** pulam at boot, reconciles per host; the client restore loop **dials+adopts, not respawns** (closes the respawn-vs-surviving-PTY race). Blip-survival is free from `HostSession`'s reconnect + the durable `kaval --stdio` daemon.
- **kill/attach** (already location-routed); **sleep/wake on the host**; **paste/upload to the host** (PR-2's scratch-write); **transcript from the host** (PR-2's transcript-read).
- The **create knob** ("On SSH host…" free-text → `location:{kind:"remote",hostId}`; works because the drv-map is **arch-keyed**, so any ssh-reachable, Nix-provisionable host dials like `kaval-tui --host`), the **`ssh` badge** + a discoverability tip, the remote-`hostId` `daemonStatus` writer, and the Nix wiring (`@kolu/surface-nix-host` + both drv maps).

- **Why whole:** a remote tile matches a local tile on **every** axis — nothing deferred, nothing wrong-host. Every prep was a local no-op, so all local e2e stay green. *(Optional UI-only split: complete-but-invisible endpoint → then the create knob.)*
- **Needs:** PR-1 · PR-2 · PR-3 · R9.1 · R7. **Not** R-pulamweb-4; **not** any further contract bump.
- **Done:** an e2e creates a remote tile and asserts EVERY local-tile axis — PTY echo; Code-tab git/diff/tree + binary preview over real remote fs/git; live agent/recency/restore badges folded via the same `fold`; kill/attach; sleep/wake, paste/upload, transcript **on the host**; kolu-restart re-adopt; reload re-dial+adopt.
