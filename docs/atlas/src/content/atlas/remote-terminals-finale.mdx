**The last leg of <Issue n={951} /> ‚Äî kolu dials remote hosts.** R8 (<PrLink pr={1594} />) shipped the right **reader-join** (a terminal is `kolu.authored ‚ãà terminalWorkspace.awareness`, joined at the reader) ‚Äî *keep that*. The question this note settles is how kolu gets the **awareness** half for a *remote* host, and that architecture is now **decided**: split **observing** from **remembering**. A **memoryless producer** runs beside each host's kaval and emits `HostObservation` events; **kolu alone folds** them, owns the memory (on `kolu.authored`), and stamps recency with **its own clock**. The full types/API/flows are in **[the awareness design](awareness-derive-store.html)**; this note is the phase plan of record. The parent [remote-terminals](remote-terminals.html) keeps the one-line rows; the model is in [the terminal model](terminal-metadata-model.html).

<Callout kind="note" title="The settled design ‚Äî and the two directions it replaced">
The awareness boundary is **observe vs. remember** ([the awareness design](awareness-derive-store.html), outcome of a grounded 3-agent debate). It supersedes two earlier attempts, both now dead: (1) the **local-pulam-PROCESS** (#1614, **CLOSED**) ‚Äî it conflated a process boundary with code decoupling and introduced a fold-clobber intrinsic to mirroring an ephemeral process; (2) **createPulam / "share the assembly"** ‚Äî it shared the *wrong thing* (the assembly still fused deriving with storing, so the foot-gun and clobber rode along to every consumer). The converged fix doesn't share the assembly; it **deletes** it: the producer is memoryless (it can't even spell memory, by type), so there is nothing to clobber and nothing to assemble. **R9¬∑lib therefore dissolves** (below).
</Callout>

### S1 ¬∑ S2 ‚Äî the awareness foundation, and why R9¬∑lib dissolves

<Phase id="S1" name="memoryless awareness producer ‚Äî observations, no seed, HostObservation only" status="todo" needs={["R6", "R4"]} />
<Phase id="S2" name="kolu's fold owns memory ‚Äî typed patches, kolu's clock, memory‚Üíauthored, event stream" status="todo" needs={["S1"]} />

**S1** rewrites the sensors into a **memoryless producer**: it emits per-field `AwarenessObservation` events (each agent/command event flagged `fromSnapshot`/`replayed`), holds no durable memory, takes no seed, and its export type `HostObservation` (cache + live) **cannot spell** the memory fields. It *deletes* the `AwarenessSink`, both `makeAwarenessSink` impls, the `AwarenessRecord`, and the apply-and-publish read-back contract. **S2** gives **kolu** the only fold: it owns the three sticky gates, returns typed `{cache, memory, live}` patches (arm-autosave *derived* from patch class, not a boolean), stamps recency with kolu's clock, and writes the memory fields onto `kolu.authored` ‚Äî so every awareness collection everywhere is `HostObservation`. The wire carries an ordered `awarenessEvents` stream, not whole values.

**Why R9¬∑lib dissolves:** createPulam was an attempt to *share the assembly* the daemon and kolu each hand-rolled. Under S1+S2 there is **no assembly** ‚Äî the producer and the fold are pure leaves, each home's loop is ~8 lines, and the producer can't write memory, so there is nothing to clobber and nothing to consolidate. The `R9¬∑lib` / createPulam phase is **removed**.

- **Needs:** R6 (the sensor library) and R4 (the pulam daemon) ‚Äî both shipped.
- **Done (S1):** the producer emits observations and touches no host store; `AwarenessSink`/`makeAwarenessSink`√ó2/`AwarenessRecord`/seed are gone. **Done (S2):** kolu's fold is the sole writer of memory (onto `kolu.authored`); the daemon serves an `awarenessEvents` stream; the write-fence, clock authority, and single-memory-writer are enforced **by type** (tests in the awareness note).

### R9.0 ‚Äî kolu runs the producer in-process, folds

<Phase id="R9.0" name="kolu runs the memoryless producer in-process and folds" status="todo" needs={["S2"]} />

kolu's local awareness is the **memoryless producer called in-process** (no daemon, no socket, no mirror), its observations fed straight into kolu's fold. kolu seeds `current` from its durable record, folds with its clock, writes the memory fields to `kolu.authored` and the rest to the `awareness` collection. The reader-join is unchanged (`composeTerminalMetadata` now reads memory from the authored half). Clobber-free **by type** (the producer can't spell memory).

- **Migration:** a **hard cutover**, no flag/fallback; plus the memory‚Üí`kolu.authored` schema move (`SCHEMA_VERSION` bump + state migration).
- **Needs:** S2.
- **Done:** kolu runs no bespoke sink/assembly; local awareness is the producer + kolu's fold; the reader-join and disk persistence hold; a fold differential test passes. Detail: [the awareness design](awareness-derive-store.html).

### R9.3 ‚Äî remote awareness = subscribe the remote event stream + the same fold

<Phase id="R9.3" name="remote awareness ‚Äî subscribe the remote producer's event stream" status="todo" needs={["S2", "R9.2", "R7"]} />

A remote host runs the **same memoryless producer** (in its pulam daemon), serving an ordered `awarenessEvents` stream of `HostObservation`. kolu subscribes over a long-lived `getHostSession` (mirrored via R7's total dual) and folds those events through the **same** fold it uses locally ‚Äî kolu seeds `current` from its durable record, stamps recency with **its** clock, writes memory to `kolu.authored`. Because kolu seeds from memory and the stream is snapshot-then-deltas, the **snapshot frame is the reconcile** ‚Äî there is no separate `reconcileRemoteSnapshot`. Only the transport differs (in-process vs ssh); the fold, the reader-join, and disk persistence are identical.

- **Needs:** S2 (the producer + fold), R9.2 (the remote PTY), R7 (the total dual).
- **Done:** a remote tile shows live awareness folded from the remote producer's stream, via the **same** fold as local; a differential test pins it equal to the producer's observed state. Detail: [the awareness design](awareness-derive-store.html).

### R9.4 ‚Äî persisted location, reconnect, adoption

<Phase id="R9.4" name="persisted location + reconnect + adoption" status="todo" needs={["R9.2"]} />

Harden the remote lifecycle: re-validate the persisted `location` on reconnect, and cover the adoption/reconcile cases.

| the remote kaval is‚Ä¶ | kolu does |
| --- | --- |
| absent | provision + spawn fresh (`adoptOrEnsure` ‚Üí ensure) |
| live ¬∑ wire-compatible | **adopt** ‚Äî connect, never kill; its PTYs reconcile in |
| live ¬∑ a build behind | adopt **+** an "update pending" nudge |
| live ¬∑ contract-skewed | recycle (kill ‚Üí respawn) ‚Äî only on a typed `DaemonContractSkewError` |
| unreachable | bounded retry, then **degraded ‚Äî never killed** |

- **Open (decide here):** *what* is re-validated on reconnect and the fail behavior. `location` is the host discriminator (`kolu.authored`); `cwd` is an observed field ‚Äî different homes. Spell out the comparison and the fail-fast (crash loudly vs re-derive); don't invent it at implementation time.
- **Needs:** R9.2.
- **Done:** a kolu-server restart re-adopts remote PTYs; each adoption-table row has a test.
