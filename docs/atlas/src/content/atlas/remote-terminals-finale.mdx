**The last leg of <Issue n={951} /> — kolu dials remote hosts.** R8 (<PrLink pr={1594} />) shipped the right **reader-join** (a terminal is `kolu.authored ⋈ terminalWorkspace.snapshots`, joined at the reader) — *keep that*. The **awareness** architecture for a remote host is **settled** — split **observing** from **remembering**: a **memoryless producer** runs beside each host's kaval and emits `TerminalSnapshot` events; **kolu alone folds** them, owns the memory (on `kolu.authored`), and stamps recency with **its own clock** ([the awareness design](awareness-derive-store.html)). What this note re-settles is the **sequence**: the remaining work is re-cut around **extracting the terminal-workflow substrate** — the receptacle that isolates the host-lifecycle volatility — with **kolu-tui as the second consumer that proves it remote-first**, then the full web tile, then the canvas.

<Callout kind="note" title="The settled design — and the two directions it replaced">
The awareness boundary is **observe vs. remember** ([the awareness design](awareness-derive-store.html), outcome of a grounded 3-agent debate). It supersedes two earlier attempts, both dead: (1) the **local-pulam-PROCESS** (#1614, **CLOSED**) — conflated a process boundary with code decoupling and introduced a fold-clobber intrinsic to mirroring an ephemeral process; (2) **createPulam / "share the assembly"** — shared the *wrong thing* (the assembly still fused deriving with storing). The converged fix **deletes** the assembly: the producer is memoryless (it can't even spell memory, by type), so there's nothing to clobber. **R9·lib therefore dissolved.**
</Callout>

<Callout kind="warn" title="What changed in this re-plan — the prep-split is dead, the substrate is the spine">
The previous version of this note sequenced **three parallel local-preserving prep PRs** (PR-1 lifecycle · PR-2 fs/git · PR-3 awareness) → one complete-everything **F-REMOTE** → R10. That split is **replaced**. The round-2 type-enforcement debate (`debates/pr1637/`) showed the lifecycle prep's real problem: routing is a *convention*, not a boundary — the local endpoint is importable, so "hard-pin local" is a writable expression a missed seam forgets. The fix isn't to thread the seam by hand (PR-1); it's to **lift the whole host-lifecycle volatility into a sealed `@kolu/*` package** both UIs embed — the [terminal substrate](sutra.html). That makes the seam-hunt a *compile error*, slims kolu-server by ~half, and turns ssh-remote into an additive endpoint. So PR-1 **dissolves into the substrate**; PR-2 (fs/git) and PR-3 (awareness rendering) **reframe** from "preps that gate remote" into **full-client surface work that lands in F-REMOTE**; and **kolu-tui** is inserted as the second consumer that proves the substrate **remote-first**. **And it's single-host:** a kolu surface shows one host at a time (host = *connection context*, switched by a dropdown), so there is no mixed local+remote canvas, no composite key, and no `HostLocation` threaded through every op — the cross-host fleet glance stays [pulam-web](pulam-web.html)'s. *(Strategic + re-evaluation debates: `debates/pr1637/`, `debates/sutra-reconsidered/`.)*
</Callout>

**S1** rewrote the sensors into a **memoryless producer**: per-field `TerminalEvent` events, no durable memory, no seed; its export type `TerminalSnapshot` (the five re-observable fields) **cannot spell** the memory fields. **S2** gave **kolu** the only fold: `fold(cur, o, ctx) → TerminalState` (`{snapshot, memory}`), last-write-wins for the five snapshot fields, stamping recency on a live agent-identity change with kolu's clock, remembering `lastAgentCommand`. ## The remaining phases

<a id="substrate"></a>

### SUBSTRATE — extract `@kolu/sutra` *(do next · the spine)*

<Phase id="SUBSTRATE" name="extract the terminal-workflow substrate — connectSutra(host) · sealed single-host dispatch · private endpoints · dial/reconnect/adoption · bare ptyId" status="next" needs={["R9.1"]} blocks={["KOLU-TUI", "F-REMOTE"]} links={[{ label: "the substrate note", href: "sutra.html" }]} />

<Svg svg={boundary} caption="The package boundary the SUBSTRATE phase creates — both UIs embed @kolu/sutra, each CONNECTED to one selected host, depending outward on the already-graduated transport (kaval · pulam · surface-nix-host · surface). The full design — API spec, the compile-error guarantee, the single-host / bare-ptyId model — is in the substrate note." />

Lift the host-lifecycle volatility into **one `@kolu/*` package both UIs embed** — scoped to **one host per connection**. You `connectSutra(host)`; per-terminal ops become `op(id)` that resolve **that connection's** endpoint **internally**; the endpoint, the resolver and the nine `*Local*` helpers go **package-private**, so "hard-pin local / bypass the host router" becomes a **compile error** (no symbol to import). A terminal's id is the **bare `ptyId`** — host is *connection context*, not a per-terminal property, so there is **no composite key and no `HostLocation` threaded through every op**. This **absorbs the manual lifecycle-threading prep** (the seam-hunt is now the compiler's job) and migrates ~**half of kolu-server** out of the web server.

- **Why whole:** a pure **local-preserving refactor + package move** — every terminal is on the local connection today, so every `op(id)` resolves to the same local endpoint; create/attach/kill/sleep/wake/adoption stay byte-identical. **No user-facing change.**
- **The full design — the single-host API, the compile-error guarantee, the bare-ptyId model, the session split, what moves vs stays — is in [the substrate note](sutra.html).**
- **Needs:** R9.1. **Blocks:** kolu-tui and the full client both embed the result.
- **Done:** the sealed single-host `TerminalSubstrate` facade + `connectSutra(host)` ship; the endpoint/resolver/`*Local*` helpers are package-private (importing one outside the package fails to compile); kolu-server drives every op through the facade with a **bare `ptyId`**; `HostLocation` no longer threads through per-terminal ops (it lives on the connection); all terminal e2e + `sleepWake.test.ts` + boot-adoption pass unchanged.

### KOLU-TUI — the second consumer, remote-first *(the de-risk milestone)*

<Phase id="KOLU-TUI" name="kolu-tui embeds the substrate and proves it remote-first — local+remote PTY · reconnect · boot re-adopt · adoption · basic awareness" status="todo" needs={["SUBSTRATE"]} blocks={["F-REMOTE"]} links={[{ label: "the kolu-tui note", href: "kolu-tui.html" }]} />

**kolu itself, in the terminal** — it **embeds** the substrate (Option B — own process, own state, *not* a thin client of a running kolu-server) and **dials one host itself**, rendering **kolu's own two-pane layout** — the **Dock rail** you switch terminals in, the **live terminal** you type into, and a **host selector** to switch which host — as a full-screen TUI on [OpenTUI](kolu-tui.html) that reuses the shared `agentProjection` folds, **one host at a time**, **with no Code browser**. It is the **second consumer that proves the substrate graduated** — and it **MUST exercise the pulam dial** for awareness, which (because awareness and fs/git ride the *same* `getHostSession({binary:"pulam"})` surface) proves **fs/git's transport for free**. It ships local+remote PTY + reconnect + boot re-dial-re-adopt + adoption + basic awareness — the **hardest, least-reversible third** of remote terminals — and is a usable product. It deliberately skips the Code tab, byte preview, paste/upload and transcript.

- **Why it's the proof, not just a product:** kaval-tui already proves raw PTY transport — kolu-tui's added value is the **kolu orchestration** both consumers share, plus the **dial coverage**. The two conditions (embed + exercise the pulam dial) are what keep it a de-risk rather than a second UI.
- **The surface, the UX prototypes, the phases (K1 embed + the Dock · K2 survive-the-link · K3 live Dock badges over the pulam dial) are in [the kolu-tui note](kolu-tui.html).**
- **Needs:** SUBSTRATE. **Blocks:** F-REMOTE inherits a *proven* substrate.
- **Done:** a remote pane is created/attached/driven/killed through the embedded substrate; the link can drop and recover; kolu-tui restart re-dials + re-adopts; the host selector switches local ↔ remote and each shows its own terminals; a remote agent's `working→awaiting` shows in the rail over the pulam dial.

### F-REMOTE — the complete web tile on the proven substrate *(nothing deferred)*

<Phase id="F-REMOTE" name="remote terminal — PTY · fs/git Code tab · byte preview · full awareness · restore · sleep/wake · paste/upload · transcript" status="todo" needs={["KOLU-TUI", "R7"]} blocks={["R10"]} />

The web/electron remote tile, lit **whole** on the substrate kolu-tui already proved. The hard constraint stands: **a remote tile is as complete as a local tile the moment a user can create one — nothing deferred.** This is where the **fs/git Code-tab procedure+pulse, the byte primitives, and the awareness framer's consumer rendering + restore UX** land — no longer preps that gate remote, but the **web surface** on a proven foundation. A remote tile becomes a full peer to a local one:

- **PTY** over the remote **kaval** (the substrate's remote endpoint; remote spawn-env from the host's `system.info`).
- **Real fs/git Code tab** + the host-aware **binary-preview byte route** over the remote **pulam** (R7 mirror) — *the transport kolu-tui already exercised; F-REMOTE adds the rendering*: rewrite the Code tab off `koluSurface`'s value-streams onto the shared surface's **procedure + `{seq}` pulse**, add the byte primitives (range-capable preview-read, scratch-write, transcript-read), re-enforce the realpath/traversal guard **on the producing host**.
- **Full awareness** — subscribe `terminalEvents` on the same mirror, fold via the **same** `fold`, keyed by the **bare `ptyId`** (host is the connection, not part of the id). Agent badge + recency + restore-target, richly rendered.
- **Session restore incl. boot re-dial-re-adopt** — reuses the substrate's per-host boot re-adopt (`connectSutra` + adopt, proven by kolu-tui); the client restore loop **dials+adopts, not respawns**; kolu re-`connectSutra`s each saved host.
- **kill/attach** (substrate-routed); **sleep/wake on the host**; **paste/upload to the host**; **transcript from the host**.
- The **host-selector dropdown** ("On SSH host…" → `connectSutra({kind:"remote",hostId})`), the **`ssh` badge** + a discoverability tip, the remote-`hostId` `daemonStatus` writer, and the Nix wiring.

- **Why whole:** a remote tile matches a local tile on **every** axis — and every substrate primitive it rides was proven by kolu-tui, so the only new risk is the *web rendering*, not the transport. All local e2e stay green.
- **Needs:** KOLU-TUI · R7. **Not** R-pulamweb-4; **not** any further contract bump beyond the byte primitives.
- **Done:** an e2e creates a remote tile and asserts EVERY local-tile axis — PTY echo; Code-tab git/diff/tree + binary preview over real remote fs/git; live agent/recency/restore badges folded via the same `fold`; kill/attach; sleep/wake, paste/upload, transcript **on the host**; kolu-restart re-adopt; reload re-dial+adopt. All local e2e green.

## Open decisions

The awareness forks are **decided** (recorded in [the awareness design](awareness-derive-store.html)): memory home = `kolu.authored`; recency = identity-only; dashboards = urgency-only; the activity predicate is one tested function. The sequencing forks are **decided** here: substrate → kolu-tui → full-client F-REMOTE → R10; the old prep-split is replaced (see the re-plan note above); R-pulamweb-4 off kolu's critical path. **Single-host is decided** (`debates/sutra-reconsidered/`): one host per surface, switched by a dropdown — the cross-host fleet glance is [pulam-web](pulam-web.html)'s, not kolu's. The id model is **decided**: a **bare `ptyId`**; host is *connection context* (`connectSutra`), not a per-terminal property — no composite key, no `HostLocation` threading ([the substrate note](sutra.html)). The awareness split is **decided**: the substrate exposes `subscribeTerminal(id)`; the **memoryless snapshot fold** — the Dock's badges via the shared `agentProjection` — rides both embedders, while the **memoryful recency baseline stays kolu-server's** (per the awareness design); the producer wire is pulam's additive `terminalEvents` stream (`3.0→3.1`), ratified at K3. The one implementation decision still open:

| reconnect re-validation semantics (folded into kolu-tui's boot re-dial-re-adopt — the first phase with a real remote) | **KOLU-TUI** | what's compared (`location` vs the remote kaval's live `cwd`) + the fail-fast behavior |
