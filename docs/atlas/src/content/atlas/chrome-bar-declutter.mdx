The original rail chose **"always two explicit columns… no collapse-when-equal"**

has stabilized — so collapse-when-equal is now on the table.

## Collapse when they agree

At rest, **one calm thing**: a single *worst-of* health dot, the version, and the
shared commit shown **once**. Hover (or click the `▸`) for the full three-way
split — nothing is lost, it just stops shouting.

The whole switch rides accessors that **already exist** — no new derivation,
no wire change:

```ts
// the resting dot: worst-of, reusing the existing wsDot / kavalDot tables.
// Branch order IS the priority: a down link wins (identity is untrustworthy
// with no server to read), then an unknown daemon stays grey — never a
// false-green before the first status yield — then a down/warming daemon,
// then the amber currency nudge, else all-clear.
const unifiedDot = (): string => {
  if (props.status !== "open") return wsDot(props.status);
  const state = daemon()?.state;
  if (!state) return kavalDot(undefined);           // grey-not-red trap kept
  if (downState() || daemonWarming()) return kavalDot(state);
  if (stale() || kavalUpdatePending()) return nudgeDot;
  return toneDot.ok;
};

// No aggregate `diverged()` predicate: each source decides for itself whether
// to fan back out, via its own <Show>. The kaval source is the one that needs
// a named test (behind, or not cleanly running); ws (props.status !== "open")
// and client (stale()) are <Show> guards inline.
const kavalDiverged = (): boolean =>
  kavalUpdatePending() || !!downState() || daemonWarming();
```

## Fan out on divergence

The collapse is a *steady-state* optimization, **never an alarm muffler**. The
moment any axis goes non-green, the offending source re-materializes inline with
its own dot, SHA and actionable chip — louder than today, not quieter. WS-down and
daemon-down stay **distinct** reds (server-unreachable ≠ daemon-dead).

## What moves where — nothing is lost
