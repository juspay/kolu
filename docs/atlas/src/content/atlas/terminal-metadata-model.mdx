**Today, a terminal's facts live in one fused record.** kolu's `koluSurface` serves a single **`terminalMetadata`** collection; its value is `Terminal = active | sleeping`. That record — `entry.meta`, held in the server's terminal registry, where *the registry is the store* — fuses **three sources of facts**: kaval's live PTY state, the `@kolu/terminal-workspace` **sensors'** derivations, and kolu's own UI. The server re-publishes the whole record on every change and persists it (minus the live overlay) at sleep. There are **no `KavalMeta` / `PulamMeta` / `KoluMeta` types** and **no compose function** — those are the names and the split the cleanup (PR1) proposes.

## The three sources

| source | fields | where they come from |
| --- | --- | --- |
| **kaval** (the PTY) | `cwd` · `title` · `foreground` · `command` · `exit` | kaval **owns** these as live `Entry` state — `cwd` from OSC 7, `title` from OSC 0/2, `command` from OSC 633;E, plus the `exit` code. **`foreground` is sampled live** (`tcgetpgrp`), not cached. (+ the write·resize·kill·I/O handle.) |
| **sensors** (`@kolu/terminal-workspace`) | `git` · `pr` · `agent` · `lastAgentCommand` · `agentSession` · `lastActivityAt` | **derived** from kaval's taps — `git` resolved from `cwd`, `pr` from `git`, `agent` detection, the *normalised* agent command (`parseAgentCommand`). Run by **kolu-server's local endpoint today, by `pulam` remotely**. |
| **kolu** (the UI author) | `location` · `themeName` · `parentId` · `canvasLayout` · `subPanel` · `rightPanel` · `intent` + `state` | authored by kolu-server/client; `location` is set once at spawn and never mutated; `state` is the `active` \| `sleeping` discriminant. |

Those map onto **two served collections** — and kolu's browser reads only one of them:

- **`awareness`** (8 fields: `cwd · git · lastAgentCommand · agentSession · lastActivityAt · pr · agent · foreground`) on **pulam's** `terminalWorkspaceSurface` — kaval-relayed (`cwd`/`foreground`/`command`) **+** sensor-derived (`git`/`pr`/`agent`). **kolu does not read this collection today.**
- **`terminalMetadata`** on **kolu's** `koluSurface` — `awareness` **⊕** `location` **⊕** the client UI fields, fused into one value. **This** is what `useTerminalMetadata` reads.

<Callout kind="note" title="Who knows what">
The sensors **relay** kaval's `cwd`/`title`/`foreground` (a process basename joined with the title tap) and **normalise** its command — kaval has no notion of "git" or "agent"; that interpretation is the sensors' job. And kolu's display reads the *fused* `terminalMetadata`, whose `cwd`/`foreground` ultimately **come from kaval** (kolu-server bridges kaval's taps straight in, and at boot treats kaval's list-entry as the authority for `cwd`/`foreground`). So `cwd` originates at kaval, not pulam.
</Callout>

## Active vs Sleeping

```text
  ACTIVE  (state:"active")                     SLEEPING  (state:"sleeping")
  ───────────────────────────                  ───────────────────────────────
  fused record, live overlay present           PTY killed · sensors torn down
  pr·agent·foreground re-derived each tick      frozen snapshot (one pr copied in)
  wire: upsert({...entry.meta}) per change      disk: SavedTerminal (overlay stripped)
```

Sleep **kills** the PTY (`releaseSleptPty` → `terminal.kill`) and tears down the sensors; **wake re-spawns** a fresh PTY and resumes the agent (`claude --resume <id>`). The "PTY survives in the daemon and gets re-adopted" path is a **different** lifecycle event — a kolu-server **restart** (`adoptLocalOrphan`), not sleep.

## Persistence — author a snapshot, don't persist the live collection

<Callout kind="note" title="persist = author (this part is real today)">
The live `terminalMetadata` collection is **push-only** — its `upsert` only notifies subscribers; **the registry is the store**, nothing in the collection hits disk. At sleep/session-save, kolu **authors** a `SavedTerminal` via `SavedTerminalSchema.parse({ ...entry.meta, id })`, which **strips the live overlay** (`pr`/`agent`/`foreground`). So disk carries the **persisted** awareness half (`cwd` · `git` · `lastAgentCommand` · `agentSession` · `lastActivityAt`) **fused with** kolu's `location` + UI fields — **not "kolu's data only."** A restored or sleeping tile shows **`cwd` / `git` from that snapshot**; **`agent` / `pr` / `foreground` are re-derived** on wake (the sleeping arm keeps one frozen `pr` copy).
</Callout>

## Identity (confirmed against the code)

```text
key  =  ( location , TerminalId )

  TerminalId   a uuid — minted by kolu-server, passed verbatim to kaval, and used
               to key BOTH collections (awareness + terminalMetadata).
  location     { kind:"local" } | { kind:"remote", hostId }   — lives in kolu's fields
```

Today the **same id value** flows through all three: kaval's `PtyId` *is* the terminal id — kolu-server mints it (`crypto.randomUUID()`) and hands it to kaval, which uses it verbatim. But the **types** still differ: `TerminalId = z.string().uuid()`, while kaval's `PtyId = string` (opaque, un-branded). PR1 brands `PtyId = TerminalId`.

## The proposed cleanup — name the sources, stop fusing server-side

<Callout kind="note" title="First principle (the target — not today)">
**A terminal has no single record — its facts have three producers.** Today they are fused server-side into one `terminalMetadata` value, re-fused on **every tick** *and* again **on disk**. The cleanup gives the three sources **names** and a **type each**, serves pulam's `awareness` as the collection kolu's browser reads, and **joins them at the reader** — so the wire carries kolu's own fields only and there is **no server-side re-fusion**. That join is *projected*, never stored; the one materialised copy is the **sleep snapshot**, which kolu already authors today.
</Callout>

```ts
// PROPOSED (PR1) — names for what already exists. None of these types exist today.

// kaval  →  KavalMeta   // kaval's live Entry state: cwd·title·foreground·command·exit (+ write·resize·kill·I/O handle)
//                       // today: ptyHost taps + PtyListEntry; PtyId is an opaque string (z.string(), not .uuid())

// pulam  →  PulamMeta = AwarenessValue   // the 8-field awareness collection (kaval-relayed + sensor-derived)
//             { cwd; git; lastAgentCommand?; agentSession?; lastActivityAt; pr; agent; foreground }

// kolu   →  KoluMeta   // kolu's own, persisted: location + themeName?·parentId?·canvasLayout?·subPanel?·rightPanel?·intent?
//                      //   + the discriminant: { state:"active" } | { state:"sleeping"; sleptAt; pr? }

type Terminal =
  | { state: "active";   id: TerminalId; /* KoluMeta + KavalMeta + PulamMeta — joined at the reader, not stored fused */ }
  | { state: "sleeping"; id: TerminalId; sleptAt; /* KoluMeta + frozen snapshot (cwd·git + resume refs + one pr) */ };
```

The real discriminant today is already `state: "active" | "sleeping"` — `ActiveTerminalSchema` / `SleepingTerminalSchema` in `@kolu/common`. PR1 keeps those names and adds the per-producer types around them.

## Plan — three phases

```text
PR1  types      name KavalMeta · PulamMeta(=AwarenessValue) · KoluMeta · keep Terminal = active|sleeping ·
                brand PtyId = TerminalId · refactor the code onto these types
P2   redo R8    serve PulamMeta(awareness) as the collection kolu's browser reads · join at the reader ·
                stop fusing entry.meta server-side · wire carries kolu's fields only
R9   remote     swap where PulamMeta is backed: local sensors → a mirrored remote awareness collection — nothing else
```

**Done-criterion a shim can't satisfy:** kolu's own browser reads the served **`awareness` (PulamMeta)** collection · the wire carries **kolu's own fields only** · **no server-side re-fusion**. None of this is true today — today is the single fused `terminalMetadata` record described above.
