**Each producer owns the type for what it makes:** kaval owns **`Pty`** (the live process), pulam owns **`Sensed`** (the live readings — cwd · git · agent · pr · foreground), kolu owns **`Owned`** (its durable UI). kolu **composes** them into **`Terminal`** — the dependency arrow points *out*. There is no separate "metadata record"; `Terminal` **is** the composition.

## Live vs Sleeping

```text
  LIVE   (PTY exists)                 SLEEPING   (PTY released)
  ────────────────────                ────────────────────────
  owned + pty + sensed                owned + snapshot   (= frozen Sensed)
  sensed projected by id              no pty · no live sensed
  disk: owned (+ restore-seeds)       disk: owned + snapshot
```

<Callout kind="note" title="First principle">
**A terminal has no metadata — `Terminal` is a composition, kept *virtual*.** The live arm is *projected* from its producers (kaval's `pty`, pulam's `sensed[id]`); nothing is stored fused. Materialise it exactly once: at **sleep** (freeze `Sensed` into a snapshot). Every other materialisation is the bug — co-owning one record (two writers → the green-dot lie, #1564) or re-fusing it server-side (PR #1594's `composeTerminalMetadata`, every tick).

**persist = author.** The live `Sensed` collection never hits disk; kolu *authors a snapshot* of it (a restore-seed, or the whole sleep snapshot), and **that** is what's persisted — so session-restore and sleeping tiles show `cwd`/`git`/`agent` from the **owned snapshot**, not from a live read. The PTY survives in the daemon; once it re-attaches, live `sensed` takes back over.

`Sensed` and `Owned` are each a `Collection` (an `Incremental`); `Terminal` is a join of two — see [surface live data](surface-live-data.html). Nothing to invent.
</Callout>

## The types (PR1)

```ts
// kaval   →   export type Pty    = …                                  // the live process handle
// pulam   →   export type Sensed = { cwd; git; agent; pr; foreground; … }   // a LIVE-ONLY collection, keyed by id, NEVER persisted

// kolu:
type Owned = { theme; layout; panels; intent; restoreSeeds }           // kolu's own · PERSISTED (incl. the snapshot)

type Terminal =
  | { state: "live";     id: TerminalId; owned: Owned; pty: Pty; sensed: Sensed }   // pty + sensed projected live, by id
  | { state: "sleeping"; id: TerminalId; owned: Owned; snapshot: Sensed }           // frozen Sensed, now kolu-owned & persisted
```

```text
illegal states become unrepresentable ✓
  · pty/sensed live only on the LIVE arm     → a sleeping terminal can't hold live state
  · snapshot lives only on SLEEPING          → a live terminal can't carry a frozen copy
  · pr (in Sensed) has ONE writer (the sensor) → frozen into snapshot at sleep, never re-authored
```

## Plan — three phases

```text
PR1  types        kaval exports Pty · pulam exports Sensed · kolu composes Terminal = Live | Sleeping · refactor onto them
P2   redo R8      serve Sensed as the collection · kolu projects it into Terminal at the reader · slim the wire ·
                  delete composeTerminalMetadata (server) · disk = Owned only
R9   remote       swap where Sensed is backed (local store → mirrored remote collection) — nothing else
```

Done-criterion a shim can't satisfy: **kolu's own browser reads the served `Sensed` collection · the wire carries `Owned` only · no server-side recompose.**
