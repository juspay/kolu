**Fundamentally a terminal is a kaval PTY.** `observed` (sensors watch the PTY) and `authored` (kolu's durable UI) are two layers on its **id** — one writer each. The "`TerminalMetadata` record" the Dock and Code tab render is their **join**, derived at the reader.

<Svg svg={diagram} caption="Three producers over one id: kaval owns the live PTY; pulam's sensors observe it (ephemeral); kolu authors the durable UI (persisted). LIVE = the join, kept virtual. SLEEP releases the PTY → there is no observed → kolu freezes the join into a snapshot (now an authored, persisted value) — the one moment the join is materialised." />

## Live vs Sleeping

```text
  LIVE   (PTY exists)                 SLEEPING   (PTY released)
  ────────────────────                ────────────────────────
  authored ⋈ observed[id]             authored + snapshot
  observed re-derives live            no PTY · no observed
  disk: authored + restore-seeds      disk: authored + snapshot
```

<Callout kind="note" title="First principle">
**A terminal has no metadata — it is a join, kept *virtual*.** Materialise it exactly once: at **sleep**. Every other materialisation is the bug — co-owning one record (two writers → the green-dot lies, #1564) or re-fusing it server-side (PR #1594's `composeTerminalMetadata`, every tick).

**persist = author.** `observed` never hits disk; to keep an observed fact, kolu *authors a snapshot* of it (a restore-seed, or the whole sleep snapshot). So everything on disk is authored; the PTY survives in the daemon (kolu re-attaches); `observed` re-derives.

`observed` and `authored` are each a `Collection` (an `Incremental`); the live view is a join of two — see [surface live data](surface-live-data.html). Nothing to invent.
</Callout>

## The types (PR1)

```ts
type Authored = { theme; layout; panels; intent; restoreSeeds }  // kolu, always, PERSISTED
type Observed = { cwd; git; agent; pr; foreground; … }           // sensors, LIVE-ONLY collection, keyed by id, NEVER persisted
type Snapshot = Authored & Frozen<Observed>                      // freeze(authored ⋈ observed) at sleep

type Terminal =
  | { state: "live";     id: TerminalId; authored: Authored }                 // observed[id] joins live · no snapshot field
  | { state: "sleeping"; id: TerminalId; authored: Authored; snapshot: Snapshot } // no live observed
```

```text
illegal states become unrepresentable ✓
  · observed is not a field of Terminal      → a sleeping terminal can't hold live observed
  · snapshot lives only on  sleeping         → a live terminal can't carry a frozen snapshot
  · frozen pr lives in snapshot, not authored → pr has ONE writer (the sink, live); frozen at sleep
```

## Plan — three phases

```text
PR1  types        Terminal = Live | Sleeping · Observed = a live-only collection · refactor the code onto them
P2   redo R8      serve observed as the awareness collection · kolu joins at the reader · slim the wire ·
                  delete composeTerminalMetadata (server) · disk = authored only
R9   remote       swap where observed is backed (local store → mirrored remote collection) — nothing else
```

Done-criterion a shim can't satisfy: **kolu's own browser reads the served `observed` collection · the wire carries `authored` only · no server-side recompose.**
