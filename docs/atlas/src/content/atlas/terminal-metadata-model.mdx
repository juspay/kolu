**Each producer owns the type for what it makes:** kaval owns **`KavalMeta`** (the live PTY — I/O · exit · the raw taps pulam reads), pulam owns **`PulamMeta`** (the 8-field `AwarenessValue` — cwd · git · agent · pr · foreground · lastActivityAt · lastAgentCommand · agentSession), kolu owns **`KoluMeta`** (its durable UI — **`location`** · themeName · parentId · canvasLayout · subPanel · rightPanel · intent). kolu **composes** them into **`Terminal`** — the dependency arrow points *out*. There is no separate "metadata record"; `Terminal` **is** the composition.

## Live vs Sleeping

```text
  LIVE   (PTY exists)                       SLEEPING   (PTY released)
  ──────────────────────────                ──────────────────────────────
  KoluMeta + KavalMeta + PulamMeta          KoluMeta + snapshot  (frozen PulamMeta)
  PulamMeta projected by id                 no KavalMeta · no live PulamMeta
  disk: KoluMeta (+ restore-seeds)          disk: KoluMeta + snapshot
```

<Callout kind="note" title="First principle">
**A terminal has no metadata — `Terminal` is a composition, kept *virtual*.** The live arm is *projected* from its producers (kaval's `KavalMeta`, pulam's `PulamMeta[id]`); nothing is stored fused. Materialise it exactly once: at **sleep** (freeze `PulamMeta` into a snapshot). Every other materialisation is the bug — co-owning one record (two writers → the green-dot lie, #1564) or re-fusing it server-side (PR #1594's `composeTerminalMetadata`, every tick).

**persist = author.** The live `PulamMeta` collection never hits disk; kolu *authors a snapshot* of it (a restore-seed, or the whole sleep snapshot), and **that** is what's persisted — so session-restore and sleeping tiles show `cwd`/`git`/`agent` from the **`KoluMeta` snapshot**, not from a live read. The PTY survives in the daemon; once it re-attaches, live `PulamMeta` takes back over.

`PulamMeta` and `KoluMeta` are each a `Collection` (an `Incremental`); `Terminal` is a join of two — see [surface live data](surface-live-data.html). Nothing to invent.
</Callout>

## The types (PR1)

```ts
// kaval  →  export type KavalMeta   // the live PTY: write·resize·kill · exit · output bytes · raw taps (cwd·title·command·foreground) pulam observes

// pulam  →  export type PulamMeta = {   // = AwarenessValue · a LIVE-ONLY collection, keyed by TerminalId, NEVER persisted
//             cwd; git: GitInfo | null; agent: AgentInfo | null; pr; foreground: Foreground | null;
//             lastAgentCommand?; agentSession?; lastActivityAt }

// kolu:
type KoluMeta = {                        // kolu's own · PERSISTED
  location;                              //   { kind:"local" } | { kind:"remote", hostId }   — which host
  themeName?; parentId?; canvasLayout?; subPanel?; rightPanel?; intent?
}

type Terminal =
  | { state: "live";     id: TerminalId; kolu: KoluMeta; kaval: KavalMeta; pulam: PulamMeta }   // kaval + pulam projected live
  | { state: "sleeping"; id: TerminalId; kolu: KoluMeta; sleptAt; snapshot: PulamMeta }         // frozen PulamMeta + sleptAt · kolu-owned & persisted
```

```text
illegal states become unrepresentable ✓
  · KavalMeta/PulamMeta live only on the LIVE arm   → a sleeping terminal can't hold live state
  · snapshot lives only on SLEEPING                  → a live terminal can't carry a frozen copy
  · pr (in PulamMeta) has ONE writer (the sensor)    → frozen into snapshot at sleep, never re-authored
```

## Identity

```text
key  =  ( KoluMeta.location , TerminalId )

  TerminalId   a uuid · the ONE id kaval (PtyId), pulam (PulamMeta key) and kolu all share
  location     which host: { kind:"local" } | { kind:"remote", hostId }  ← lives in KoluMeta, not PulamMeta

kaval & pulam key by TerminalId · kolu qualifies with location (only matters at R9, remote).
PR1 also brands  PtyId = TerminalId  — kaval's id IS the terminal id.
```

## Plan — three phases

```text
PR1  types        kaval exports KavalMeta · pulam exports PulamMeta · kolu composes Terminal = Live | Sleeping · refactor onto them
P2   redo R8      serve PulamMeta as the collection · kolu projects it into Terminal at the reader · slim the wire ·
                  delete composeTerminalMetadata (server) · disk = KoluMeta only
R9   remote       swap where PulamMeta is backed (local store → mirrored remote collection) — nothing else
```

Done-criterion a shim can't satisfy: **kolu's own browser reads the served `PulamMeta` collection · the wire carries `KoluMeta` only · no server-side recompose.**
