Most work earns a dedicated Atlas note — a bug with a root cause, a feature with
a design, an analysis with measurements. **Sundry is the holding pen for
everything too small for that**: a one-line idea, a chore, a "we should look at
this" that would be overkill as its own note. Jot it down here so it isn't lost,
and when an item grows a real shape, graduate it to its own note.

## What belongs here

An item belongs in Sundry when it's **real work but not yet note-sized** — there
is no design to argue, no root cause to diagnose, no measurement to report. The
bar is deliberately low: a single row with a *what* and a *goal* is enough. The
moment an item needs more than that — a diagram, a repro, a plan, a debate — it
has outgrown Sundry (see *Graduating an item*).

## The list

| Item | What | Goal |
| --- | --- | --- |
| **Unify the two Dock activity dots** | Each Dock row shows two separate dots — terminal activity and agent activity — with dead space on the left. | Merge them into one indicator (e.g. terminal-activity dot enveloping the agent dot) so a single glance conveys *overall* activity. This stays honest when agent activity is wrongly paused but the terminal is still live, and reclaims the wasted left margin. |
| **Distinguish the active terminal in a split** | When a tile is split, nothing visually marks which pane — main or split — currently has focus. | Add a clear visual cue (border, header tint, or similar) that distinguishes the active terminal from the inactive one at a glance. |

## Graduating an item

When a Sundry item earns a design, a repro, or a plan, it has stopped being
sundry. Create a real note under its true index (`parents: [bug]`, `[feature]`,
`[analysis]`, or `[reference]`), move the substance there, and **delete the row**
here — keeping a one-line prose pointer only if the link is worth following.
Sundry should stay short: a long list is a signal that items are being hoarded
instead of either done or promoted.
