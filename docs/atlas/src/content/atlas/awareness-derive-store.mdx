**The de-entanglement that makes R9 mostly evaporate.** Every hard thing in [the finale](remote-terminals-finale.html) ‚Äî the two assemblers, the createPulam boundary, the fold-clobber ‚Äî traces to **one** complect: the sensor engine *derives* awareness by *mutating a host record it also reads back as memory*. Split **observing** from **remembering** and it dissolves. This note pins the **types, the API, and the flows** so the boundary can be evaluated before any code; every claim cites the code it restructures.

<Callout kind="note" title="Provenance ‚Äî this is a settled design, not a sketch">
This supersedes the createPulam / shared-leaf direction of the finale's R9¬∑lib + R9.0, **and** the earlier "S1+S2 / seed-the-engine" draft of this very note. It is the converged outcome of a grounded, adversarial 3-agent design debate held to the perfection bar (the record: `debates/awareness-architecture/conclusion.md`). The forks the debate exposed are **decided** (below), not left open.
</Callout>

After the [`architecture-first-principles`](/architecture-first-principles) skill was distilled from this design, all three debate agents ran it back over the plan adversarially (`debates/awareness-architecture/05.*.md`). <Callout kind="note" title="Foundations ‚Äî a fold over a stream, not shared mutable state">
The shape of this design is a deliberate move off **shared mutable state** onto **an immutable fold over a stream of observations**. The vocabulary that makes it legible (and that a future reader should reason in):
- **Observations are a log/stream.** The producer's interface is a stream of events ("observed git = X"), not a writable record. It is **memoryless** ‚Äî *effectful but memoryless* (it runs watchers/timers), **not** a pure function.
- **The fold is a pure reduce.** `fold(current, observation) ‚Üí delta` is a reducer; the served state is a left-fold (scan) over the stream, and the gate functions (`shouldBumpRecencyForAgentChange`, `agentSessionToPersist`, ‚Ä¶) are pure. The *fold* and the *diff* are duals ‚Äî `fold : (state, event) ‚Üí state'`, `diff : (state, state') ‚Üí event`.
- **Immutability.** The fold builds new values (`{ ...current, ...patch }`); it never mutates a shared record. That in-place mutation *was* the original complect (the knot below).
- **A materialized read model (CQRS flavor).** The whole-value `awareness` collection is a derived projection of folding the stream ‚Äî the stream is the input, the snapshot the output.
- **One writer; consumer-owned clock.** Memory has a single writer (kolu's fold), and recency is stamped by the **consumer's** clock ‚Äî a remote producer's wall clock is never imported as ordering truth (distributed-clock discipline).

**Two honest limits, so the anchor doesn't over-reach:** (1) the producer is *memoryless and effectful* ‚Äî "pure" applies to the **fold and gates**, not the producer; (2) this is **fold-to-state, not classic event sourcing** ‚Äî we persist the derived *state* (on disk), not the event log. The stream is ephemeral; a reconnect re-folds from a snapshot frame. Borrow the *concepts* (reduce ¬∑ immutability ¬∑ materialized view ¬∑ log), not the machinery (a durable, replayable log).
</Callout>

## The knot today ‚Äî observing is fused with remembering

The sensor engine does **not** return awareness. It takes a host `AwarenessSink` + a per-terminal `AwarenessRecord`, **mutates `record.meta` through the sink**, and **reads `record.meta` back** as its own prior state. `sensors.ts:139-162` calls this the load-bearing *apply-and-publish* contract ‚Äî a sink that publishes *without* mutating the record "would silently defeat every dedup/transition gate."

So `record.meta` plays **two roles at once**:

- the engine's **prior-state** for its gates ‚Äî command dedup `record.meta.lastAgentCommand` (`sensors.ts:410`), publish-if-changed `agentInfoEqual(record.meta.agent, ‚Ä¶)` (`sensors.ts:516`), recency `shouldBumpRecencyForAgentChange(‚Ä¶, record.meta.lastActivityAt)` (`sensors.ts:519`), session dedup `agentSessionToPersist(record.meta.agentSession, ‚Ä¶)` (`sensors.ts:526`);
- the host's **stored value** ‚Äî kolu aliases `record.meta = getTerminal(id)!.awareness` (`local.ts:700-704`); `pulam` points it at a cache (`daemon.ts:209-217`).

<Svg svg={before} caption="Today: the sensor set and the host's storage are the SAME object. record.meta is both the engine's prior-state (read back by the dedup/recency/session gates) and the host's stored value (mutated in place via the sink). That fusion is the cause of everything downstream: **two assemblers** (`local.ts:278` vs `pulam/src/hooks.ts:39`), the **`AwarenessSink` foot-gun** (mutate-before-publish or defeat the gates), and the **fold-clobber class** (a fresh/ephemeral derivation overwrites remembered history ‚Äî the defect that killed the local-pulam-process #1614 and that the createPulam loop re-risked).

## The cut: three categories of fact, by *authority*

The debate's keystone. A terminal's awareness is not one bag ‚Äî it is three categories with three owners:

```ts
// ‚îÄ‚îÄ Durable observation ‚Äî TRUE-when-dead; survives the PTY, persisted to avoid a restore flash.
//    pr re-files HERE: a forge fact keyed by the branch, NOT the PTY (true-when-dead, like git). ‚îÄ‚îÄ
type AwarenessCache = { cwd: string; git: GitInfo | null; pr: PrResult };

// ‚îÄ‚îÄ Live ‚Äî LIE-when-dead; dies with the PTY process, so never persisted ‚îÄ‚îÄ
type AwarenessLive  = { agent: AgentInfo | null; foreground: Foreground | null };

// ‚îÄ‚îÄ kolu's DURABLE INTERPRETATION of what it observed ‚Äî irrecoverable, kolu's alone ‚îÄ‚îÄ
type AwarenessMemory = { lastActivityAt: number; agentSession?: AgentSessionRef; lastAgentCommand?: string };

// What a host PRODUCER may export. It literally cannot spell memory.
type HostObservation = AwarenessCache & AwarenessLive;

// kolu's folded state. The three authorities are NESTED, not intersected ‚Äî so the
// host slice published to the awareness collection is `current.cache`+`current.live`,
// which structurally CANNOT carry a memory field. `KoluAwareness` is NOT a subtype
// of `HostObservation`, so `awareness.upsert(id, current)` is a COMPILE error, not a
// value that happens to be zod-stripped at runtime (round-5 P4 fix).
type KoluAwareness = { cache: AwarenessCache; live: AwarenessLive; memory: AwarenessMemory };
```

`HostObservation` is the load-bearing type: a producer ‚Äî local or remote ‚Äî **cannot express** `lastActivityAt` / `agentSession` / `lastAgentCommand`, so a remote snapshot **cannot clobber kolu's memory**, by the compiler, not a comment. The persisted/live partition (`schema.ts:139-176`) was too coarse; the right structure is **two orthogonal axes** (below). And `KoluAwareness` **nests** the three rather than intersecting them: an intersection (`HostObservation & AwarenessMemory`) is a structural *supertype* of `HostObservation`, so the host slice would type-check with memory riding along ‚Äî the leak is only caught by a runtime zod strip. Nesting makes the host projection (`{ ...current.cache, ...current.live }`) carry no memory field *by construction*.

<Callout kind="note" title="Three categories = two axes (refined from a follow-up debate)">
The three categories aren't a flat partition ‚Äî they're the intersection of **two orthogonal axes**:
- **Authority** ‚Äî *who may write it*: **host-producible** (`HostObservation` = cache ‚à™ live) vs **kolu-only** (memory). This is the **keystone** cut ‚Äî the producer can't spell memory.
- **Truth-when-dead** ‚Äî *is the last value true after the PTY dies*: **true-when-dead** (cache, memory ‚Äî persist as stale-but-real) vs **lie-when-dead** (live ‚Äî drop; persisting it fabricates a referent). The cache/live line is a *sub-split inside `HostObservation`*, forced by the autosave firehose ‚Äî it must be a **type** so a live patch can't arm autosave.

So it's **2 primary + 1 sub-split** (a 2√ó2 with one empty cell: kolu-only ‚à© lie-when-dead = ‚àÖ). That's why **"persist everything"** fails (1 category revives the firehose + a stale lie) and **"durable vs ephemeral"** fails (2 categories merge cache+memory, so the producer's type would contain memory ‚Üí the keystone breaks). A field may move **cache‚Üîlive** freely (both are host-producible, same wire), but **nothing may cross `HostObservation ‚Üî Memory`**. `pr` is **true-when-dead** (a forge fact keyed by the branch, not the PTY), so it re-files to **cache** ‚Äî which deletes the frozen-`pr` special case (`local.ts:871`, the sleeping-arm `pr?`). `agent`/`foreground` are **lie-when-dead** (their process dies with the PTY), so they stay live ‚Äî persisting them is a lie *and* the agent firehose.
</Callout>

<Callout kind="note" title="Decided ‚Äî memory rides kolu.authored">
The three `AwarenessMemory` fields **move onto `kolu.authored`** (the record that already carries `location` ¬∑ theme ¬∑ the active/sleeping discriminant). Consequence: the `awareness` collection **everywhere** ‚Äî kolu's own and every host producer's ‚Äî carries only `HostObservation`. "Memory on an awareness collection" becomes globally unspellable; kolu's fold is the sole writer of memory, into the authored half. Cost: a schema move + `SCHEMA_VERSION` bump + a state migration; `composeTerminalMetadata` reads memory from the authored half. (Reader-join shape unchanged ‚Äî still `authored ‚ãà awareness` at the client and at disk.)
</Callout>

## S1 ‚Äî the producer is memoryless

It owns transient *derivation* working state (the agent watcher, `latestInfo`, the last-emitted live mirror, `currentAgent`, the git‚ÜíPR wire, the screen-scrape poll, the adapter registry ‚Äî all re-seeded empty each start) and emits **per-field observations**. It takes **no seed** and touches no host store.

```ts
type AwarenessObservation =
  | { kind: "cwd";        cwd: string }
  | { kind: "git";        git: GitInfo | null }                       // fold derives recentRepo from this
  | { kind: "pr";         pr: PrResult }
  | { kind: "foreground"; foreground: Foreground | null }
  | { kind: "agent";      agent: AgentInfo | null }                  // already publish-if-changed
  | { kind: "commandRun"; command: string; replayed: boolean };      // fold derives recentAgent from this

function startAwarenessEngine(
  id: TerminalId,
  inputs: {
    pid: number;
    cwd: string;                                          // spawn-time cwd; later cwd flows via signals
    signals: AwarenessSignals;                            // the four taps ‚Äî UNCHANGED
    readScreenText?: (tailLines: number) => Promise<string>;  // pure input for screen-scrape (#905)
    log: Logger;
  },
  emit: (o: AwarenessObservation) => void,
): () => void;
```

**`replayed`** is the one provenance flag that stays on the observation, and it must be a **required** field (a flag-less event is a compile error, so it can't be dropped at a wire boundary). It is genuine *content* ‚Äî kaval's tap (`sensors.ts:103-106`) reports whether a command mark is a replay or a live run; the producer does not guess it. **`fromSnapshot` is gone** (round-5 P2¬∑P5 fix): "is this a first-frame re-observation or a live change" is **not a property of the event** ‚Äî the *same* `agent = X` observation is a snapshot if it lands in kolu's initial burst and a delta if it lands later; only **kolu's** position relative to its own subscribe boundary differs. A producer-asserted boolean kolu *trusts* for recency is the wall-clock-skew defect relocated from a `number` to a `boolean` ‚Äî a producer that mislabels it on a restart re-observation bumps recency spuriously. So liveness rides a **frame-level `snapshot|delta` discriminator kolu owns** (see [the wire](#the-wire-observations-not-whole-values)), never a per-event flag. *(codex's stronger variant ‚Äî split `commandRun` into `commandRunLive | commandRunReplay` event kinds so the bad combination can't be written either ‚Äî is compatible; `replayed`-as-content is the lighter call we take.)*

**Deleted:** the `AwarenessSink` interface, *both* `makeAwarenessSink` impls, the `AwarenessRecord` type, the apply-and-publish read-back contract ‚Äî and any `seed`. The git‚ÜíPR channel and the adapter registry were always engine-internal and stay.

## S2 ‚Äî kolu remembers; the fold owns the gates and the clock

kolu ‚Äî the **sole** owner of memory ‚Äî seeds `current` from its durable record, folds each observation, and stamps recency with **kolu's** clock. The fold returns **typed patches by authority**; arm-autosave is *derived* from a **change in the persisted projection** (cache ‚à™ memory), never a patch's mere presence ‚Äî so a no-op tick can't arm it and the `~150 ms` agent firehose cannot reach disk.

```ts
type FoldDelta = {
  cache?:  Partial<AwarenessCache>;    //  cwd ¬∑ git ¬∑ pr    ‚Üí persisted (arms autosave when a VALUE changes)
  memory?: Partial<AwarenessMemory>;   //  the 3 mem fields  ‚Üí persisted, lands on kolu.authored
  live?:   Partial<AwarenessLive>;     //  agent ¬∑ foreground ‚Üí quiet (never persisted)
  recentRepo?: { root: string; name: string };   // fold-derived MRU side-effects
  recentAgent?: string;
};

// Liveness + clock are kolu's, passed as VALUES, not a thunk the reducer may fire (round-5 P2):
//   live  ‚Äî derived from the FRAME PHASE kolu received (delta ‚áí true, snapshot ‚áí false), not a per-event flag
//   at    ‚Äî kolu samples its clock once, at intake, before calling the fold
type FoldCtx = { live: boolean; at: number };

function fold(current: KoluAwareness, o: AwarenessObservation, ctx: FoldCtx): FoldDelta {
  switch (o.kind) {
    case "cwd": return { cache: { cwd: o.cwd } };
    case "git": return { cache: { git: o.git },
                         recentRepo: o.git ? { root: o.git.mainRepoRoot, name: o.git.repoName } : undefined };
    case "pr":         return { cache: { pr: o.pr } };   // pr is true-when-dead (like git) ‚Äî persisted, not live
    case "foreground": return { live: { foreground: o.foreground } };

    case "agent": {
      if (agentInfoEqual(current.live.agent, o.agent)) return {};   // no redundant re-emit ‚Äî empty, not { live }
      const memory: Partial<AwarenessMemory> = {};
      // RECENCY (decided: identity-only). Bump iff a LIVE identity change ‚Äî the SAME trigger as session-persist.
      // No `lastActivityAt` read, no restore-caveat heuristic: the FRAME PHASE (ctx.live) replaces it.
      const identityChanged = current.live.agent?.kind !== o.agent?.kind
                           || current.live.agent?.sessionId !== o.agent?.sessionId;
      if (ctx.live && identityChanged) memory.lastActivityAt = ctx.at;          // kolu's clock, never the host's
      const session = agentSessionToPersist(current.memory.agentSession, o.agent);  // #1495, unchanged pure fn
      if (session) memory.agentSession = session;
      return { live: { agent: o.agent }, memory };
    }

    case "commandRun":
      return current.memory.lastAgentCommand === o.command
        ? { recentAgent: o.replayed ? undefined : o.command }
        : { memory: { lastAgentCommand: o.command }, recentAgent: o.replayed ? undefined : o.command };
  }
}
```

```ts
// The host recipe. kolu is the ONLY home that folds memory. A producer (the daemon) runs the
// memoryless engine and serves HostObservation events ‚Äî it cannot emit a memory patch at all.
// `persisted` is the true-when-dead projection (cache ‚à™ memory); arming compares it across the
// fold step, so an empty/no-op patch ‚Äî e.g. an agent token tick that changes no memory field ‚Äî
// produces an EQUAL projection and DOES NOT arm. The firehose can't reach disk, by construction.
const persisted = (a: KoluAwareness) => ({ ...a.cache, ...a.memory });

function watch(key: HostTerminalKey, terminal): () => void {
  let current = seed(terminal);                 // kolu seeds from its durable record (the SAME join as adoptedAwareness)
  const stop = subscribeAwareness(key, terminal, (frame) => {           // frame: snapshot | delta | gap (see the wire)
    if (frame.phase === "gap") { current = reseed(key); return; }       // a detected gap re-snapshots ‚Äî never a silent diverge
    const ctx: FoldCtx = { live: frame.phase === "delta", at: Date.now() };  // kolu samples its clock ONCE, here
    for (const o of frame.observations) {
      const before = current;
      const d = fold(current, o, ctx);
      current = apply(current, d);              // nested merge: { cache:{...}, live:{...}, memory:{...} } ‚Äî new value, no mutation
      if (!shallowEqual(persisted(before), persisted(current))) armAutosave();  // VALUE change, not patch presence
      if (d.recentRepo)  trackRecentRepo(d.recentRepo.root, d.recentRepo.name);
      if (d.recentAgent) trackRecentAgent(d.recentAgent);
    }
    awareness.upsert(key, { ...current.cache, ...current.live });   // HostObservation ‚Äî memory STRUCTURALLY absent
    authored.updateMemory(key, current.memory);                    // the ONE memory writer (a narrowed mutator)
  });
  return () => { stop(); /* late events after stop are ignored ‚Äî the post-teardown no-op guard */ };
}
```

<Callout kind="note" title="Why this is clobber-free, with no seed">
The producer **cannot write memory** (`HostObservation` has no memory fields), so nothing it emits can overwrite a remembered value. kolu seeds `current` from its own durable record and folds; a memory field the engine hasn't re-observed is simply never in a patch, so it survives. The old "seed the engine + restore-caveat" machinery is gone: recency reads no prior `lastActivityAt`; the **frame phase** kolu received (`ctx.live` ‚Äî a fact kolu owns, not a producer flag) ‚Äî not a `lastActivityAt>0` heuristic ‚Äî tells the fold a first-frame re-observation from a real change. (`shouldBumpRecencyForAgentChange`'s restore caveat at `agentRecency.ts:31` is **deleted**, not ported ‚Äî and it fixes a latent bug: a genuinely-new agent started after a prior one finished is no longer wrongly suppressed.)
</Callout>

## The wire ‚Äî observations, not whole values

The collection serving a whole `AwarenessValue` (`surface.ts:119-124`) is **retired** as the awareness primitive. Producers serve an **ordered `awarenessEvents` stream** (host-scoped, snapshot-then-deltas, subscribe-before-serialize ‚Äî the attach contract, `router.ts:162-169`, and the `streaming.md` rule) of `HostObservation` events. The wire carries the fold's *input*, not its lossy *output*: a whole value drops the `replayed` fact and the clock context. There is **no `observedAt`** on the wire (a clock-skew temptation).

The stream is **framed**, not a bare event flow ‚Äî the frame is what carries provenance and order so the *observation* vocabulary can stay field-level (round-5 P3 fix, and the `streaming.md`-prescribed shape):

```ts
type AwarenessFrame =
  | { phase: "snapshot"; observations: HostObservation[] }            // kolu folds these with ctx.live = false
  | { phase: "delta";    seq: number; observations: HostObservation[] }  // ctx.live = true; seq is monotonic per subscription
  | { phase: "gap";      afterSeq: number };                          // a detected coalescing/overflow gap ‚Üí kolu re-snapshots
```

Two prior "decided" calls **reverse** here, because the lens showed each one re-admitted a defect the design forbids:
- **An explicit `snapshot` frame kind** (was: "no explicit snapshot event kind"). The snapshot-vs-live fact is **kolu's** subscription-phase fact, not a per-event boolean a producer asserts and kolu trusts. Putting it on the *frame* lets kolu derive `ctx.live` from the frame it received ‚Äî its own knowledge ‚Äî and **deletes `fromSnapshot`**.
- **A per-subscription monotonic `seq`** (was: "`seq` only if a stream needs explicit ordering"). This stream **drives memory**, and the coalescing rule below *promises* gap-signaling ‚Äî a promise with no mechanism. `seq` is the distinguisher (precedent: `RepoChangePulse.seq`, `surface.ts:58-67`): kolu asserts contiguity, and a hole forces a `gap` frame ‚Üí re-snapshot, instead of a silently divergent fold. A per-subscription counter is **not** a wall clock, so it imports nothing and the `observedAt` objection doesn't apply to it.

kolu consumes this framed stream through one `subscribeAwareness(key, ‚Ä¶)` seam regardless of transport: **locally** it wraps the in-process `startAwarenessEngine` (the initial emit burst becomes the `snapshot` frame, later emits become `delta`s); **remotely** it is the ssh stream verbatim. So the framing ‚Äî not the engine ‚Äî is where snapshot/live and `seq` live, and the engine stays a pure memoryless emitter.

The payoff: **kolu's fold is one function for local and remote, and `reconcileRemoteSnapshot` disappears.** kolu seeds `current` from its durable record, then folds the producer's events ‚Äî whether they arrive **in-process** (localhost) or **over ssh** (remote). The snapshot frame meets a pre-seeded `current`; because it arrives as `phase: "snapshot"`, kolu folds it with `ctx.live = false` and so suppresses a spurious recency bump; the cache fields overwrite the cached ones. `adoptedAwareness` (`local.ts:340`), wake, remote-connect, and remote-reconnect are the **same** "seed from memory, then fold" ‚Äî `orphanAwareness` (`local.ts:368`) is the degenerate zero-memory seed.

<Callout kind="warn" title="Scale + coalescing (decided)">
Volume is tens‚Äìhundreds of small events/sec at realistic fleet sizes ‚Äî the firehose is already coalesced at source by `agentInfoEqual` (`sensors.ts:516`), and it is the same count the browser already fans out. **Coalescing rule:** the latest-wins display fields (`agent` detail, `pr`, `foreground`) may coalesce; the **discrete marks may not** ‚Äî `commandRun` (the `replayed` MRU bit), the agent **identity/state** transition (drives recency *and* the `awaiting_user` alert), and `git`/`cwd` MRU edges. Default: coalesce nothing beyond `agentInfoEqual`; if scale ever bites, split `agent` into a lossless `agentState` lane and a coalescible `agentDetail` lane. On buffer overflow the mechanism is now concrete (round-5 P3): the producer emits a `phase: "gap"` frame and kolu re-snapshots ‚Äî the monotonic `seq` lets kolu *detect* the hole rather than fold a silently divergent stream.
</Callout>

## How R9 collapses

A "host" is now *dial a kaval ‚Üí per terminal: run the memoryless producer ‚Üí serve its observation stream*. **kolu** is the one consumer that remembers: it folds every host's stream (localhost in-process, remotes over ssh) into a host-keyed aggregate. "Two homes" was an artifact ‚Äî there is **one producer per host; localhost is the degenerate (in-process) host**.

## Decided ‚Äî the forks the debate closed

- **The activity rule** (the one product judgment): a terminal's recency (`lastActivityAt`) bumps **only on a live agent-identity change** ‚Äî it starts, finishes, or a new session appears ‚Äî **not** on `thinking‚Üîawaiting` state flips or token churn. Chosen as the simplest gate (recency shares its trigger with session-persistence and reads no prior memory). The "needs you" alert is unaffected ‚Äî it rides live `agent.state`/urgency, not recency.
- **`pr` re-files to cache** (follow-up refinement): `pr` is true-when-dead (a forge fact keyed by the branch, not the PTY), so it moves `AwarenessLive ‚Üí AwarenessCache` and persists like `git` ‚Äî deleting the frozen-`pr` special case (`local.ts:871`, the sleeping-arm `pr?`) and showing the badge immediately on restore. `agent`/`foreground` stay live (lie-when-dead).
- **Memory home ‚Üí `kolu.authored`** (so every awareness collection is `HostObservation`). The fold is its **one** writer through a narrowed `authored.updateMemory(key, AwarenessMemory)` mutator (round-5 P3) ‚Äî mirroring the existing narrowed `updateClientMetadata` (`metadata.ts:166`) ‚Äî so "the fold writes only memory; client code never writes memory" is typed, not a convention the two writers must remember.
- **Dashboards (pulam-tui / pulam-web) ‚Üí urgency-only** ‚Äî they drop recency-based fleet ordering (a secondary tiebreak) and need no fold; they read `HostObservation` + the urgency projection. Because `HostObservation` has no `lastActivityAt`, a dashboard that still imports recency **fails to compile** ‚Äî the break is a compile gate, not a note (split `compareAgents` into `compareAgentUrgency` (host-safe) and a kolu-only recency tiebreak).
- **Host-scoped aggregation key ‚Äî a branded `HostTerminalKey` type** (round-5 P3¬∑P5), not the prose pair `(HostLocation, TerminalId)`: kaval ids are opaque (`ptyHostSurface.ts:87`), so a bare `Map<TerminalId, ‚Ä¶>` still *spells* the cross-host orphan collision. kolu intake pairs the host-local `TerminalId` with its `HostLocation` into `HostTerminalKey` **before** the fold can run; aggregate maps, fold state, and authored records are all keyed by it, so a fleet view cannot alias two hosts' non-kolu-minted PTY ids.

## Phasing & migration

- **S1 ‚Üí S2 ‚Üí R9.** S1 (the memoryless producer) is the one real lift ‚Äî rewriting the freshness-critical sensors from mutate-via-sink to emit-observations, with the existing sensor tests as the safety net; it *deletes* the sink √ó2, the record, the read-back contract, the seed, and the clobber. S2 (kolu's fold + the memory‚Üíauthored move + the framed `awarenessEvents` stream) follows, and **deletes the awareness *mutator* APIs too, not only the sink** (round-5 P1): `mutateAwarenessPersisted` / `mutateAwarenessLive` (`terminal-registry.ts:177-197`) and the `(m) => { m.x = ‚Ä¶ }` callbacks they feed (`metadata.ts:124-159`) are the surviving mutable *place* ‚Äî the commit seam **replaces** the registry value (`entry.awareness = next`) instead of handing a caller a mutable object. After both, R9¬∑lib is gone, R9.0 is an ~8-line loop, and R9.3 is subscribe-and-fold.
- **Hard cutover, no knobs** (the fail-fast rule). Memory is single-writer *by type* (`HostObservation` can't spell it; `KoluAwareness` nests it so the host slice can't carry it), the clock is kolu's *by type* (the producer has no memory field to stamp) and a sampled *value* not a thunk the reducer may fire, and the autosave fence rides a **persisted-value change** ‚Äî `arms` is honestly "by a value check," not "by type," since `Partial<T>` always admits `{}` (round-5 P4: the one place the design can't reach a pure type, made explicit rather than overclaimed).
- **What it retires:** createPulam; the `AwarenessSink`; the seed and the restore-caveat heuristic; `reconcileRemoteSnapshot`; the "two homes, two assemblers" framing; and the fold-clobber as a *class*. #1614 stays dead for the right reason ‚Äî the in-process producer can't clobber, so no local pulam *process* is needed.

## The one thing no type settles ‚Äî and its containment

Exactly one judgment is a product choice, not a type: **which agent transition counts as "activity."** It's decided (identity change, above) and lives in **one** tested function with **no bypass** (memory patches are constructable only via the gate). Almost everything else is unspellable *by type*; the single honest exception is the autosave fence, which rides a **persisted-value change** because `Partial<T>` admits `{}` ‚Äî so it gets a test, not a type. **Tests that make that real:**

- *type* ‚Äî a `HostObservation` cannot contain `lastActivityAt`/`agentSession`/`lastAgentCommand`; `KoluAwareness` is not assignable to `HostObservation` (publishing `current` whole to the awareness collection is a compile error); no `reconcileRemoteSnapshot(AwarenessValue, AwarenessValue)` API exists.
- *the headline fence* ‚Äî an `agent` token tick that changes **no** persisted field (same identity, same session ‚Äî the ~150 ms firehose) folds to an **equal** persisted projection and **does not** arm autosave. Remove the value-change check and this test fails (the firehose reaches disk).
- a replayed `commandRun` updates `lastAgentCommand` but fires **no** recent-agent bump.
- a `null‚Üídetected` agent in a **`delta`** frame bumps recency; the same in a **`snapshot`** frame (producer restart) does **not** ‚Äî the frame phase kolu received, not a producer flag, decides; there is no `fromSnapshot` field to mislabel.
- a producer clock a year ahead never reorders kolu's recency (kolu samples its own clock as a value at intake; the wire has no `observedAt`).
- a dropped delta (a hole in `seq`) yields a `gap` frame and a re-snapshot ‚Äî never a silently divergent fold.
- the same id on two `HostLocation`s keeps two aggregate entries (a bare `TerminalId` can't index the fleet ‚Äî `HostTerminalKey` is required).
- debouncing a discrete `commandRun`/`cd` mark is a failing contract test ‚Äî only display state may coalesce.
