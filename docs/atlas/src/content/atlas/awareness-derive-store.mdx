**The de-entanglement that makes R9 mostly evaporate.** Every hard thing in [the finale](remote-terminals-finale.html) — the two assemblers, the createPulam boundary, the fold-clobber — traces to **one** complect: the sensor engine *derives* awareness by *mutating a host record it also reads back as memory*. Split **observing** from **remembering** and it dissolves. This note pins the **types, the API, and the flows** so the boundary can be evaluated before any code; every claim cites the code it restructures.

<Callout kind="note" title="Provenance — this is a settled design, not a sketch">
This supersedes the createPulam / shared-leaf direction of the finale's R9·lib + R9.0, **and** the earlier "S1+S2 / seed-the-engine" draft of this very note. It is the converged outcome of a grounded, adversarial 3-agent design debate held to the perfection bar (the record: `debates/awareness-architecture/conclusion.md`). The forks the debate exposed are **decided** (below), not left open.
</Callout>

## The knot today — observing is fused with remembering

The sensor engine does **not** return awareness. It takes a host `AwarenessSink` + a per-terminal `AwarenessRecord`, **mutates `record.meta` through the sink**, and **reads `record.meta` back** as its own prior state. `sensors.ts:139-162` calls this the load-bearing *apply-and-publish* contract — a sink that publishes *without* mutating the record "would silently defeat every dedup/transition gate."

So `record.meta` plays **two roles at once**:

- the engine's **prior-state** for its gates — command dedup `record.meta.lastAgentCommand` (`sensors.ts:410`), publish-if-changed `agentInfoEqual(record.meta.agent, …)` (`sensors.ts:516`), recency `shouldBumpRecencyForAgentChange(…, record.meta.lastActivityAt)` (`sensors.ts:519`), session dedup `agentSessionToPersist(record.meta.agentSession, …)` (`sensors.ts:526`);
- the host's **stored value** — kolu aliases `record.meta = getTerminal(id)!.awareness` (`local.ts:700-704`); `pulam` points it at a cache (`daemon.ts:209-217`).

<Svg svg={before} caption="Today: the sensor set and the host's storage are the SAME object. record.meta is both the engine's prior-state (read back by the dedup/recency/session gates) and the host's stored value (mutated in place via the sink). Because deriving writes the store, a fresh derivation can clobber remembered state, two sinks must each wire the assembly, and 'share the assembly' (createPulam) is the only way two homes can reuse the sensing." />

That fusion is the cause of everything downstream: **two assemblers** (`local.ts:278` vs `pulam/src/hooks.ts:39`), the **`AwarenessSink` foot-gun** (mutate-before-publish or defeat the gates), and the **fold-clobber class** (a fresh/ephemeral derivation overwrites remembered history — the defect that killed the local-pulam-process #1614 and that the createPulam loop re-risked).

## The cut: three categories of fact, by *authority*

The debate's keystone. A terminal's awareness is not one bag — it is three categories with three owners:

```ts
// ── Re-derivable from the host's taps + filesystem; persisted only to avoid a restore flash ──
type AwarenessCache = { cwd: string; git: GitInfo | null };

// ── Live; never persisted ──
type AwarenessLive  = { pr: PrResult; agent: AgentInfo | null; foreground: Foreground | null };

// ── kolu's DURABLE INTERPRETATION of what it observed — irrecoverable, kolu's alone ──
type AwarenessMemory = { lastActivityAt: number; agentSession?: AgentSessionRef; lastAgentCommand?: string };

// What a host PRODUCER may export. It literally cannot spell memory.
type HostObservation = AwarenessCache & AwarenessLive;
```

`HostObservation` is the load-bearing type: a producer — local or remote — **cannot express** `lastActivityAt` / `agentSession` / `lastAgentCommand`, so a remote snapshot **cannot clobber kolu's memory**, by the compiler, not a comment. The persisted/live partition (`schema.ts:139-176`) was too coarse; this is its sharpening into *re-derivable cache* vs *irrecoverable memory*.

<Callout kind="note" title="Decided — memory rides kolu.authored">
The three `AwarenessMemory` fields **move onto `kolu.authored`** (the record that already carries `location` · theme · the active/sleeping discriminant). Consequence: the `awareness` collection **everywhere** — kolu's own and every host producer's — carries only `HostObservation`. "Memory on an awareness collection" becomes globally unspellable; kolu's fold is the sole writer of memory, into the authored half. Cost: a schema move + `SCHEMA_VERSION` bump + a state migration; `composeTerminalMetadata` reads memory from the authored half. (Reader-join shape unchanged — still `authored ⋈ awareness` at the client and at disk.)
</Callout>

## S1 — the producer is memoryless

It owns transient *derivation* working state (the agent watcher, `latestInfo`, the last-emitted live mirror, `currentAgent`, the git→PR wire, the screen-scrape poll, the adapter registry — all re-seeded empty each start) and emits **per-field observations**. It takes **no seed** and touches no host store.

```ts
type AwarenessObservation =
  | { kind: "cwd";        cwd: string }
  | { kind: "git";        git: GitInfo | null }                       // fold derives recentRepo from this
  | { kind: "pr";         pr: PrResult }
  | { kind: "foreground"; foreground: Foreground | null }
  | { kind: "agent";      agent: AgentInfo | null; fromSnapshot: boolean }  // already publish-if-changed
  | { kind: "commandRun"; command: string; replayed: boolean };      // fold derives recentAgent from this

function startAwarenessEngine(
  id: TerminalId,
  inputs: {
    pid: number;
    cwd: string;                                          // spawn-time cwd; later cwd flows via signals
    signals: AwarenessSignals;                            // the four taps — UNCHANGED
    readScreenText?: (tailLines: number) => Promise<string>;  // pure input for screen-scrape (#905)
    log: Logger;
  },
  emit: (o: AwarenessObservation) => void,
): () => void;
```

The two carried flags are load-bearing and must be **required** fields (a flag-less event is a compile error, so they can't be dropped at a wire boundary): **`fromSnapshot`** distinguishes the producer's first-frame re-observation (on (re)start) from a genuine live change; **`replayed`** distinguishes a replayed command mark from a live one.

**Deleted:** the `AwarenessSink` interface, *both* `makeAwarenessSink` impls, the `AwarenessRecord` type, the apply-and-publish read-back contract — and any `seed`. The git→PR channel and the adapter registry were always engine-internal and stay.

## S2 — kolu remembers; the fold owns the gates and the clock

kolu — the **sole** owner of memory — seeds `current` from its durable record, folds each observation, and stamps recency with **kolu's** clock. The fold returns **typed patches by authority**; arm-autosave is *derived* from a non-empty cache/memory patch, never a hand-set boolean — so the `~150 ms` agent firehose cannot reach disk, *by type*.

```ts
type FoldDelta = {
  cache?:  Partial<AwarenessCache>;    //  cwd · git          → arms autosave
  memory?: Partial<AwarenessMemory>;   //  the 3 memory fields → arms autosave (lands on kolu.authored)
  live?:   Partial<AwarenessLive>;     //  pr · agent · fg     → quiet
  recentRepo?: { root: string; name: string };   // fold-derived MRU side-effects
  recentAgent?: string;
};
const arms = (d: FoldDelta) => !!(d.cache || d.memory);   // misrouting a field (agent→memory) is a COMPILE error

function fold(current: KoluAwareness, o: AwarenessObservation, now: () => number): FoldDelta {
  switch (o.kind) {
    case "cwd": return { cache: { cwd: o.cwd } };
    case "git": return { cache: { git: o.git },
                         recentRepo: o.git ? { root: o.git.mainRepoRoot, name: o.git.repoName } : undefined };
    case "pr":         return { live: { pr: o.pr } };
    case "foreground": return { live: { foreground: o.foreground } };

    case "agent": {
      if (agentInfoEqual(current.agent, o.agent)) return { live: { agent: o.agent } };   // no redundant re-emit
      const memory: Partial<AwarenessMemory> = {};
      // RECENCY (decided: identity-only). Bump iff a LIVE identity change — the SAME trigger as session-persist.
      // No `lastActivityAt` read, no restore-caveat heuristic: the `fromSnapshot` fact replaces it.
      const identityChanged = current.agent?.kind !== o.agent?.kind
                           || current.agent?.sessionId !== o.agent?.sessionId;
      if (!o.fromSnapshot && identityChanged) memory.lastActivityAt = now();   // kolu's clock, never the host's
      const session = agentSessionToPersist(current.agentSession, o.agent);    // #1495, unchanged pure fn
      if (session) memory.agentSession = session;
      return { live: { agent: o.agent }, memory };
    }

    case "commandRun":
      return current.lastAgentCommand === o.command
        ? { recentAgent: o.replayed ? undefined : o.command }
        : { memory: { lastAgentCommand: o.command }, recentAgent: o.replayed ? undefined : o.command };
  }
}
```

```ts
// The host recipe. kolu is the ONLY home that folds memory. A producer (the daemon) runs a
// memory-free `foldObservation` that returns ONLY HostObservation — it cannot return a memory patch.
function watch(id: TerminalId, terminal): () => void {
  let current = seed(terminal);                 // kolu seeds from its durable record (the SAME join as adoptedAwareness)
  const stop = startAwarenessEngine(id, { pid, cwd: current.cwd, signals, readScreenText, log }, (o) => {
    const d = fold(current, o, () => Date.now());      // kolu's clock, always
    current = apply(current, d);                       // {...current, ...cache, ...memory, ...live}; memory lands on authored
    publish(current);                                  // awareness collection serves HostObservation; authored serves memory
    if (arms(d)) armAutosave();                        // terminals:dirty — derived from patch class, never asserted
    if (d.recentRepo)  trackRecentRepo(d.recentRepo.root, d.recentRepo.name);
    if (d.recentAgent) trackRecentAgent(d.recentAgent);
  });
  return () => { stop(); /* late events after stop are ignored — the post-teardown no-op guard */ };
}
```

<Callout kind="note" title="Why this is clobber-free, with no seed">
The producer **cannot write memory** (`HostObservation` has no memory fields), so nothing it emits can overwrite a remembered value. kolu seeds `current` from its own durable record and folds; a memory field the engine hasn't re-observed is simply never in a patch, so it survives. The old "seed the engine + restore-caveat" machinery is gone: recency reads no prior `lastActivityAt`; the `fromSnapshot` flag — not a `lastActivityAt>0` heuristic — tells the fold a first-frame re-observation from a real change. (`shouldBumpRecencyForAgentChange`'s restore caveat at `agentRecency.ts:31` is **deleted**, not ported — and it fixes a latent bug: a genuinely-new agent started after a prior one finished is no longer wrongly suppressed.)
</Callout>

## The wire — observations, not whole values

The collection serving a whole `AwarenessValue` (`surface.ts:119-124`) is **retired** as the awareness primitive. Producers serve an **ordered `awarenessEvents` stream** (host-scoped, snapshot-then-deltas, subscribe-before-serialize — the attach contract, `router.ts:162-169`, and the `streaming.md` rule) of `HostObservation` events. The wire carries the fold's *input*, not its lossy *output*: a whole value drops the `replayed`/`fromSnapshot` facts and the clock context. There is **no `observedAt`** on the wire (a clock-skew temptation) and **no explicit `snapshot` event kind** (the snapshot frame is just a burst of per-field events).

The payoff: **kolu's fold is one function for local and remote, and `reconcileRemoteSnapshot` disappears.** kolu seeds `current` from its durable record, then folds the producer's events — whether they arrive **in-process** (localhost) or **over ssh** (remote). The snapshot frame meets a pre-seeded `current`; the `fromSnapshot` flag suppresses a spurious recency bump; the cache fields overwrite the cached ones. `adoptedAwareness` (`local.ts:340`), wake, remote-connect, and remote-reconnect are the **same** "seed from memory, then fold" — `orphanAwareness` (`local.ts:368`) is the degenerate zero-memory seed.

<Callout kind="warn" title="Scale + coalescing (decided)">
Volume is tens–hundreds of small events/sec at realistic fleet sizes — the firehose is already coalesced at source by `agentInfoEqual` (`sensors.ts:516`), and it is the same count the browser already fans out. **Coalescing rule:** the latest-wins display fields (`agent` detail, `pr`, `foreground`) may coalesce; the **discrete marks may not** — `commandRun` (the `replayed` MRU bit), the agent **identity/state** transition (drives recency *and* the `awaiting_user` alert), and `git`/`cwd` MRU edges. Default: coalesce nothing beyond `agentInfoEqual`; if scale ever bites, split `agent` into a lossless `agentState` lane and a coalescible `agentDetail` lane. On buffer overflow: signal a gap and force a fresh snapshot — never silently collapse.
</Callout>

## How R9 collapses

A "host" is now *dial a kaval → per terminal: run the memoryless producer → serve its observation stream*. **kolu** is the one consumer that remembers: it folds every host's stream (localhost in-process, remotes over ssh) into a host-keyed aggregate. "Two homes" was an artifact — there is **one producer per host; localhost is the degenerate (in-process) host**.

## Decided — the forks the debate closed

- **The activity rule** (the one product judgment): a terminal's recency (`lastActivityAt`) bumps **only on a live agent-identity change** — it starts, finishes, or a new session appears — **not** on `thinking↔awaiting` state flips or token churn. Chosen as the simplest gate (recency shares its trigger with session-persistence and reads no prior memory). The "needs you" alert is unaffected — it rides live `agent.state`/urgency, not recency.
- **Memory home → `kolu.authored`** (so every awareness collection is `HostObservation`).
- **Dashboards (pulam-tui / pulam-web) → urgency-only** — they drop recency-based fleet ordering (a secondary tiebreak) and need no fold; they read `HostObservation` + the urgency projection.
- **Host-scoped aggregation key `(HostLocation, TerminalId)`** — a fleet view cannot alias two hosts' non-kolu-minted (orphan-adopted) PTY ids.

## Phasing & migration

- **S1 → S2 → R9.** S1 (the memoryless producer) is the one real lift — rewriting the freshness-critical sensors from mutate-via-sink to emit-observations, with the existing sensor tests as the safety net; it *deletes* the sink ×2, the record, the read-back contract, the seed, and the clobber. S2 (kolu's fold + the memory→authored move + the `awarenessEvents` stream) follows. After both, R9·lib is gone, R9.0 is an ~8-line loop, and R9.3 is subscribe-and-fold.
- **Hard cutover, no knobs** (the fail-fast rule). The write-fence is kept *by type* (the `{cache, memory, live}` patches), memory is single-writer *by type* (`HostObservation` can't spell it), and the clock is kolu's *by type* (the producer has no memory field to stamp).
- **What it retires:** createPulam; the `AwarenessSink`; the seed and the restore-caveat heuristic; `reconcileRemoteSnapshot`; the "two homes, two assemblers" framing; and the fold-clobber as a *class*. #1614 stays dead for the right reason — the in-process producer can't clobber, so no local pulam *process* is needed.

## The one thing no type settles — and its containment

Exactly one judgment is a product choice, not a type: **which agent transition counts as "activity."** It's decided (identity change, above) and lives in **one** tested function with **no bypass** (memory patches are constructable only via the gate). Everything else is unspellable by type. **Tests that make that real:**

- *type* — a `HostObservation` cannot contain `lastActivityAt`/`agentSession`/`lastAgentCommand`; no `reconcileRemoteSnapshot(AwarenessValue, AwarenessValue)` API exists.
- a replayed `commandRun` updates `lastAgentCommand` but fires **no** recent-agent bump.
- a `null→detected` agent as a **live** event bumps recency; the same in the **snapshot frame** (producer restart) does **not** — the `fromSnapshot` fact replaced the `lastActivityAt>0` heuristic.
- a producer clock a year ahead never reorders kolu's recency.
- the same id on two `HostLocation`s keeps two aggregate entries.
- debouncing a discrete `commandRun`/`cd` mark is a failing contract test — only display state may coalesce.
