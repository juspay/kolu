## UI

- Extract reusable UI into SolidJS components (one component per file in `packages/client/src/`).
- Follow the [frontend-design skill](https://github.com/anthropics/claude-code/blob/main/plugins/frontend-design/skills/frontend-design/SKILL.md) for UI design decisions — bold intentionality over generic aesthetics.

## SolidJS Patterns

- **State per domain**: Extract shared state into `useXxx.ts` modules (singleton pattern — create store once, cache at module level). Keep App.tsx as a thin layout shell.
- **Commands stay declarative**: `commands.ts` registers commands — it should not contain async RPC calls, error handling, or multi-step workflows. Those belong in `useXxx.ts` handlers. Commands just call `deps.handleFoo()`.
- **Components own their behavior**: If a component has a keyboard shortcut or toggle state, it should manage that internally (always mounted if needed), not leak it to the parent.
- **`createStore` over `createSignal<Record>`**: For keyed state (e.g. per-terminal metadata), use `createStore` from `solid-js/store` for fine-grained per-key reactivity.
- **`@solid-primitives`**: Prefer community primitives (`makePersisted`, `createResizeObserver`, `makeEventListener`) over hand-rolled equivalents.
- **Props stay reactive**: Never destructure props. Access via `props.xxx`. Pass accessors when needed.
- **Memos for multi-consumer derivations**: Use `createMemo` when 2+ reactive contexts read the same derived value. Use plain functions for single-consumer or trivial derivations.
- **Surface bound primitives for server streams**: Use `app.cells.X.use(...)`, `app.collections.X.use(...)`, `app.streams.X.use(inputFn)`, and `app.events.X.use(inputFn, handler, { onError })` from `wire.ts`'s `surfaceClient` for streaming server state. The framework owns snapshot+deltas, retry, and reconcile-vs-assign. For a SURFACE-scoped raw streaming RPC that doesn't fit a primitive (a bulk snapshot feed), use `client.rawStream(name, proc, input, { onItem })` — it enrols the stream into `client.health()` structurally (and throws if driven outside a reactive owner), so a raw stream can't silently escape the health fact. For a stream that ALREADY owns its `pending`/`error` (a `createSubscription` over a surface procedure), join it with `client.enroll(name, sub)`. The bare `unenrolledStreamCall` lives at `@kolu/surface/client` (NOT the `@kolu/surface/solid` barrel) and is only for a stream that is NOT a surface subscription — a ROOT RPC outside any surface (the terminal `attach`), where you enrol by hand or deliberately carve it out: `unenrolledStreamCall(client.X.Y, input, { signal, onRetry })`. For one-shot RPC calls, use plain `client.*` calls with `createSignal` or `createResource`.
- **A mirror's liveness rides in the fact**: `client.health().live` is now the FULL conjunction — transport-live ∧ every readiness cell's predicate (sub errors stay in `subs`). A cell becomes a readiness gate via `CellSpec.liveWhen?: (v) => boolean` (e.g. surface-nix-host's `connection` cell declares `liveWhen: (v) => v.state === "connected"`); `surfaceClient` opens an eager standing sub for each, so a `mirroredSurface`'s connection-liveness leg folds into `live` BY CONSTRUCTION. Read the complete fact — NEVER hand-AND `connection.state === "connected"`.
- **Paint the connection dot from the fact**: Render the status dot with `<HostStatusPip health={…}>` from `@kolu/surface/solid/HostStatusPip` (its own entry, like `<SurfaceGate>` — not the JSX-free `@kolu/surface/solid` barrel). Its green is fact-only (default `gateStatus(h) === "ready"`, shared with `<SurfaceGate>`); NEVER color a status dot from a raw cell `.state`.
- **Socket-backed clients use `connectSurface` / `connectSurfaces`**: They wire the always-on heartbeat / half-open watchdog in, so `health().live` catches a silent half-open (laptop sleep / wifi roam). The half-open-blind path is **unspellable** over a websocket — and unspellable for real, not by convention: `surfaceClient`/`surfaceClients` THROW unless handed a watchdog-backed `LiveSignal`, and the only minter is `createLiveSignal` (`@kolu/surface/solid`), which wires the watchdog first. The brand symbol AND its stamping function are module-private to `createLiveSignal`'s file — `brandLiveSignal` is **not exported anywhere** — so no `import` can forge a brand, and there is **no `heartbeat: false`** on the minting path to mint one without a watchdog. So a bare `{ live: () => true }` (or an open/close-only `() => socketStatus() === "live"`) is *refused*, and there is no reachable way to turn it into a `LiveSignal`. Hand-building `surfaceClient + websocketLink` is fine ONLY when you mint the `{ live }` with `createLiveSignal` (kolu's combined `wire.ts`, the minimal examples) — never a hand-rolled signal. A `directLink`/`stdioLink` can't half-open, so it needs no `{ live }`. When two layers watch one socket (an admin socket whose `connectSurfaces` owns the watchdog AND a `<SurfaceAppProvider>` lifecycle over it), the NON-brand layer (`createServerLifecycle` / the provider) opts ITS watchdog out with `heartbeat: false` — that's watchdog-ownership coordination, never a blind brand.
- **`mapArray` for dynamic per-entity subscriptions**: When the set of subscriptions is driven by a reactive list (e.g., per-terminal metadata), use `mapArray` to create subscriptions. SolidJS handles lifecycle — each item gets its own reactive owner, automatically disposed when removed from the list. (`useCollection` already does this internally; reach for `mapArray` directly only when composing outside the framework.)
- **Plain client calls for mutations**: No mutation wrappers needed. Call `client.terminal.create(...)` directly and handle errors with `.catch((err: Error) => toast.error(...))`. Server pushes update subscriptions automatically.
